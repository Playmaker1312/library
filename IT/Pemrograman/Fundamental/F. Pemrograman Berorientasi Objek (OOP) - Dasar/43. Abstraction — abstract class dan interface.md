## 1. Definisi Singkat

**Abstraction (Abstraksi)** adalah konsep menyembunyikan kerumitan proses di balik layar dan hanya menampilkan tombol atau antarmuka (_interface_) yang benar-benar penting dan mudah digunakan oleh pengguna.

Dalam kode, abstraksi sering diwujudkan melalui dua cara:

- **Abstract Class:** Sebuah cetakan "setengah jadi". Anda tidak bisa mencetak objek langsung dari _Abstract Class_. Ia hanya berfungsi sebagai kerangka dasar yang **wajib** dilanjutkan dan diselesaikan oleh _Child Class_-nya.
    
- **Interface:** Sebuah murni "kontrak kesepakatan". Interface hanya berisi daftar nama metode (tanpa isi/logika di dalamnya). Siapapun (Class manapun) yang menandatangani kontrak ini, **wajib** membuat isi dari metode-metode tersebut sendiri.
    

## 2. Analogi: Menyetir Mobil dan Kontrak Kerja

- **Abstraksi (Menyetir Mobil):** Saat Anda menyetir, Anda hanya dihadapkan pada **Setir, Pedal Gas, dan Rem** (Ini adalah _Interface_ / Antarmuka). Anda cukup menginjak gas, mobil akan melaju. Anda tidak perlu tahu (dan tidak mau tahu) seberapa rumit busi memercikkan api, atau bagaimana injektor menyemprotkan bensin di dalam mesin. Kerumitan mesin **diabstraksi** (disembunyikan) dari Anda.
    
- **Abstract Class (Kerangka Kendaraan):** Di pabrik, insinyur membuat rancangan "Kendaraan Abstrak". Kendaraan Abstrak ini belum jelas bentuknya, sehingga tidak bisa dijual ke konsumen. Ia baru bisa dijual setelah rancangan ini diturunkan (diwariskan) dan dirakit menjadi wujud spesifik seperti "Mobil SUV" atau "Truk".
    

## 3. Poin-Poin Penting

- **Fokus pada "Apa", bukan "Bagaimana":** Abstraksi fokus memberi tahu _apa_ yang bisa dilakukan oleh objek, tanpa harus menjelaskan _bagaimana_ cara kerjanya.
    
- **Standarisasi Ketat:** Interface dan Abstract Class memaksa _programmer_ dalam tim besar untuk patuh pada aturan baku. Jika kontraknya bilang "Setiap sistem pembayaran harus punya metode `prosesDana()`", maka _programmer_ tidak bisa seenaknya memberi nama `bayarUang()`.
    
- **Keterbatasan JavaScript Murni:** Perlu dicatat, JavaScript murni (ES6) **tidak memiliki** fitur _Abstract Class_ dan _Interface_ secara bawaan seperti Java atau C#. Namun, para _developer_ JS sering **menyimulasikannya**, atau mereka menggunakan **TypeScript** (versi JS yang lebih canggih) untuk mendapatkan fitur ini.
    

## 4. Contoh Implementasi

Mari kita lihat bagaimana _programmer_ menyimulasikan **Abstract Class** dalam **JavaScript** murni menggunakan pelemparan _Error_ (agar class tersebut dilarang untuk dicetak langsung):

JavaScript

```
// 1. ABSTRACT CLASS (Cetakan Setengah Jadi)
class BentukGeometri {
    constructor() {
        // Mencegah class ini dicetak langsung dengan kata kunci 'new'
        if (this.constructor === BentukGeometri) {
            throw new Error("Class BentukGeometri adalah Abstrak! Tidak bisa dicetak langsung.");
        }
    }

    // Metode Abstrak (Metode kosong yang WAJIB diisi oleh Child Class)
    hitungLuas() {
        throw new Error("Metode 'hitungLuas()' WAJIB dibuat oleh subclass-nya!");
    }
}

// 2. SUBCLASS (Menyelesaikan rancangan setengah jadi tadi)
class Persegi extends BentukGeometri {
    constructor(sisi) {
        super();
        this.sisi = sisi;
    }

    // Mematuhi kontrak: Kita wajib membuat isi dari hitungLuas()
    hitungLuas() {
        return this.sisi * this.sisi;
    }
}

// --- MARI KITA UJI ---

// Jika kita nekat mencetak Abstract Class:
// let bentukAneh = new BentukGeometri(); // ❌ ERROR: Tidak bisa dicetak langsung!

// Cara yang benar (Mencetak Subclass yang sudah jadi):
let kotakKu = new Persegi(5);
console.log("Luas Persegi: " + kotakKu.hitungLuas()); // ✅ Output: Luas Persegi: 25
```

_(Catatan: Jika Anda nanti belajar **TypeScript**, Anda cukup menulis `abstract class BentukGeometri` atau `interface Kendaraan`, yang jauh lebih rapi dan resmi)._

## 5. Koneksi dengan Web Development

Abstraksi adalah alasan mengapa aplikasi web modern bisa dihubungkan dengan berbagai layanan lain dengan sangat mudah:

1. **TypeScript di Frontend (React/Angular):** Di industri profesional, UI dikembangkan menggunakan TypeScript yang sangat mengandalkan `Interface`. Saat membuat tabel profil _user_, _developer_ mendefinisikan `interface User { nama: string, umur: number }`. Jika ada rekan tim mencoba memasukkan data `alamat`, program akan menolaknya karena menyalahi kontrak _Interface_.
    
2. **Konsep API (Application Programming Interface):** Istilah API yang sangat sering Anda dengar aslinya adalah penerapan Abstraksi! Saat aplikasi Anda menggunakan "Google Maps API", Google hanya memberikan Anda antarmuka berupa satu baris perintah: `tampilkanPeta(koordinat)`. Google menyembunyikan jutaan baris kode satelit dan server mereka dari Anda.
    
3. **Database Drivers di Backend:** Sebuah _framework backend_ sering memiliki _Interface_ `Database` yang mewajibkan metode `simpanData()`. Berkat abstraksi ini, _developer_ bisa dengan mudah menukar mesin _database_ dari MySQL ke MongoDB, asalkan _driver_ baru tersebut mematuhi kontrak `simpanData()` yang sama.
