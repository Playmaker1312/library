Mari kita bahas pilar kedua dari Pemrograman Berorientasi Objek (OOP) yang akan membuat Anda berhenti menulis kode yang sama berulang-ulang. Ini adalah rahasia bagaimana perusahaan teknologi besar mengelola jutaan baris kode dengan sangat efisien.

---

## 1. Definisi Singkat

**Inheritance (Pewarisan)** adalah konsep di mana sebuah _Class_ baru diciptakan dengan mengambil (mewarisi) seluruh atribut dan metode dari _Class_ lain yang sudah ada.

- _Class_ asal yang mewariskan sifat disebut **Parent Class / Superclass**.
    
- _Class_ baru yang menerima warisan disebut **Child Class / Subclass**.
    
- **Single Inheritance** berarti sebuah _Child Class_ hanya boleh mewarisi sifat dari **satu** _Parent Class_ saja secara langsung (tidak boleh punya dua ayah/ibu sekaligus).
    

Dalam proses pewarisan ini, anak bisa mengubah atau menimpa metode milik orang tuanya agar sesuai dengan kebutuhannya sendiri. Ini disebut **Method Overriding**.

## 2. Analogi: Karyawan Biasa dan Manajer

Bayangkan Anda sedang membuat sistem HRD untuk sebuah perusahaan:

- Anda membuat cetakan **Class Karyawan** (Parent). Semua karyawan pasti punya "Nama", "Gaji Pokok", dan bisa melakukan tindakan "Absen Pagi()".
    
- Suatu hari, bos meminta Anda membuat sistem untuk **Manajer**. Manajer pada dasarnya adalah karyawan juga, kan? Mereka punya Nama, Gaji Pokok, dan harus Absen Pagi. Tetapi, Manajer punya tambahan fasilitas: "Tunjangan Jabatan" dan tindakan "Pimpin Rapat()".
    
- Daripada Anda menulis ulang _Class_ Manajer dari nol (mengetik ulang Nama, Gaji, dan Absen), Anda cukup membuat **Class Manajer yang "mewarisi" Class Karyawan**. Anda tinggal menambahkan bagian yang berbeda saja (Tunjangan dan Rapat).
    

## 3. Poin-Poin Penting

- **Prinsip DRY (Don't Repeat Yourself):** Pewarisan mencegah Anda melakukan _copy-paste_ kode. Jika ada perubahan pada cara "Absen Pagi", Anda cukup mengubahnya di _Parent Class_, dan ratusan _Child Class_ akan otomatis mengikuti perubahan tersebut.
    
- **Kata Kunci `extends`:** Di JavaScript (dan Java), ini adalah kata ajaib untuk menghubungkan _Child_ ke _Parent_. (Contoh: `class Manajer extends Karyawan`).
    
- **Kata Kunci `super()`:** Jika _Child Class_ memiliki _Constructor_ sendiri, ia **wajib** memanggil `super()` di baris pertamanya. `super()` bertugas memanggil _Constructor_ milik _Parent_ untuk memastikan atribut dasarnya (seperti Nama dan Gaji) disiapkan terlebih dahulu sebelum atribut tambahannya diproses.
    
- **Method Overriding:** Jika _Parent_ punya fungsi `bekerja()`, _Child_ boleh membuat fungsi `bekerja()` dengan nama yang sama persis untuk **menimpa** fungsi milik _Parent_-nya.
    

## 4. Contoh Implementasi

Mari kita lihat keajaiban pewarisan dan _overriding_ ini menggunakan **JavaScript**:

JavaScript

```
// 1. SUPERCLASS (Parent)
class Karyawan {
    constructor(nama, gaji) {
        this.nama = nama;
        this.gaji = gaji;
    }

    // Metode milik Parent
    bekerja() {
        console.log(`${this.nama} sedang mengerjakan tugas biasa.`);
    }

    absen() {
        console.log(`${this.nama} telah melakukan absensi.`);
    }
}

// 2. SUBCLASS (Child) - Menggunakan kata 'extends'
class Manajer extends Karyawan {
    
    constructor(nama, gaji, departemen) {
        // WAJIB panggil super() untuk mengirim 'nama' dan 'gaji' ke Karyawan
        super(nama, gaji); 
        
        // Atribut khusus yang HANYA dimiliki Manajer
        this.departemen = departemen;
        this.tunjangan = 5000000; 
    }

    // METHOD OVERRIDING: Menimpa metode bekerja() milik Parent
    bekerja() {
        console.log(`${this.nama} sedang memimpin rapat di departemen ${this.departemen}.`);
    }

    // Metode khusus Manajer
    pecatKaryawan(namaKaryawan) {
        console.log(`${this.nama} memecat ${namaKaryawan}. Jangan ditiru ya!`);
    }
}

// --- MARI KITA UJI ---

let stafBiasa = new Karyawan("Andi", 4000000);
let bosBesar = new Manajer("Budi", 15000000, "IT");

// Karyawan biasa
stafBiasa.absen();   // Output: Andi telah melakukan absensi.
stafBiasa.bekerja(); // Output: Andi sedang mengerjakan tugas biasa.

console.log("-------------------");

// Manajer (Bisa melakukan apa yang Karyawan bisa, PLUS kemampuannya sendiri)
bosBesar.absen();    // Output: Budi telah melakukan absensi. (Dapat warisan dari Parent!)
bosBesar.bekerja();  // Output: Budi sedang memimpin rapat di departemen IT. (Hasil Overriding!)
bosBesar.pecatKaryawan("Joko"); // Output: Budi memecat Joko...
```

## 5. Koneksi dengan Web Development

Dalam **Web Development**, _Inheritance_ adalah fondasi dari banyak kerangka kerja (_framework_) modern:

1. **React Component (Zaman Dulu):** Jika Anda melihat kode React versi lawas, Anda akan melihat `class ProfilUser extends React.Component`. Artinya, komponen web Anda mewarisi ratusan baris kode rahasia dari React (seperti cara me-_render_ HTML ke layar), sehingga Anda cukup fokus menulis tampilan antarmukanya saja.
    
2. **Sistem Akun (Roles):** _Backend Developer_ selalu membuat struktur `class User` (punya email & password). Lalu membuat `class PremiumUser extends User` (bisa unduh video resolusi tinggi) dan `class Admin extends User` (bisa hapus akun orang lain).
    
3. **Membangun Elemen UI Khusus:** _Frontend Developer_ mungkin membuat cetakan standar `class BaseButton` (punya ukuran, animasi klik). Lalu mereka membuat `class DangerButton extends BaseButton`, yang mewarisi semua animasinya, tetapi me-_override_ warna tombolnya menjadi merah menyala.
