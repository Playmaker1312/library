## 1. Definisi Singkat

**Class (Kelas)** adalah cetakan biru (_blueprint_) atau template konseptual yang mendefinisikan karakteristik dan perilaku suatu entitas.

**Object (Objek)** adalah wujud nyata (_instance_) yang diciptakan dari cetakan biru tersebut. Proses menciptakan objek dari sebuah _class_ disebut dengan **Instansiasi (Instantiation)**.

## 2. Analogi: Arsitek dan Perumahan

Bayangkan Anda adalah seorang arsitek perumahan:

- **Class:** Anda menggambar **Denah Rumah (Blueprint)** di atas kertas. Denah ini menyatakan bahwa rumah tersebut _harus_ punya 2 kamar, 1 garasi, dan 1 pintu utama. Namun, Anda tidak bisa tidur di dalam denah kertas tersebut. Denah itu hanyalah konsep abstrak.
    
- **Object:** Kontraktor menggunakan denah Anda untuk membangun **Rumah Fisik** di Blok A1. Rumah ini nyata, bisa disentuh, dan Anda bisa tidur di dalamnya. Rumah di Blok A1 ini adalah sebuah _Object_.
    
- Kontraktor bisa menggunakan satu denah yang sama untuk membangun Rumah di Blok A2, Blok A3, dan seterusnya. Masing-masing rumah adalah objek yang mandiri (jika lampu di rumah A1 putus, lampu di rumah A2 tetap menyala), meskipun semuanya berasal dari _Class_ (denah) yang sama.
    

## 3. Poin-Poin Penting

Di dalam sebuah _Class_, terdapat tiga komponen utama yang wajib Anda pahami:

- **Property / Attribute:** Ini adalah _variabel_ yang diletakkan di dalam _class_. Mewakili sifat atau karakteristik objek (misal: warna cat, nama pemilik).
    
- **Method:** Ini adalah _fungsi_ yang diletakkan di dalam _class_. Mewakili tindakan yang bisa dilakukan objek tersebut (misal: buka pintu, nyalakan lampu).
    
- **Constructor:** Ini adalah blok kode khusus di dalam _class_ yang **otomatis dijalankan pertama kali** saat sebuah objek baru diciptakan. Fungsinya seperti pabrik yang merakit pengaturan awal (seperti memberi nama pemilik saat penyerahan kunci rumah).
    
- **Keyword `this`:** Kata kunci ajaib yang berarti "objek ini sendiri". Digunakan di dalam _class_ untuk membedakan antara variabel milik objek dan variabel parameter biasa.
    

## 4. Contoh Implementasi

Mari kita buat _Class_ dan mencetak _Object_ menggunakan **JavaScript** modern (ES6):

JavaScript

```
// 1. MEMBUAT CLASS (Cetakan Biru)
class Mobil {
    
    // Constructor: Fungsi yang otomatis jalan saat objek dicetak
    constructor(merkMobil, warnaMobil) {
        // 'this' merujuk pada objek yang sedang dibuat saat itu
        this.merk = merkMobil;   // Property
        this.warna = warnaMobil; // Property
        this.bensin = 100;       // Nilai default untuk semua mobil baru
    }

    // Method: Tindakan yang bisa dilakukan mobil
    nyalakanMesin() {
        console.log(`Brumm! Mesin mobil ${this.merk} berwarna ${this.warna} menyala.`);
    }

    jalan(jarak) {
        this.bensin = this.bensin - jarak;
        console.log(`${this.merk} berjalan. Sisa bensin: ${this.bensin}%`);
    }
}

// 2. INSTANSIASI OBJECT (Mencetak wujud nyata menggunakan kata kunci 'new')
let mobilBudi = new Mobil("Toyota", "Hitam");
let mobilSiti = new Mobil("Honda", "Merah");

// 3. MENGGUNAKAN OBJECT
// Kedua mobil punya identitas dan data bensin yang terpisah secara independen!
mobilBudi.nyalakanMesin(); // Output: Brumm! Mesin mobil Toyota berwarna Hitam menyala.
mobilBudi.jalan(20);       // Output: Toyota berjalan. Sisa bensin: 80%

mobilSiti.nyalakanMesin(); // Output: Brumm! Mesin mobil Honda berwarna Merah menyala.
console.log(mobilSiti.bensin); // Output: 100 (Bensin Siti tidak berkurang karena yang jalan mobil Budi)
```

## 5. Koneksi dengan Web Development

Ketika Anda membuat aplikasi web, Anda dikelilingi oleh instansiasi objek setiap harinya:

1. **Browser APIs:** Pernahkah Anda ingin menampilkan tanggal hari ini di web? Anda harus menulis `let hariIni = new Date();`. Di sini, Anda sedang mencetak objek baru (`hariIni`) dari cetakan bawaan JavaScript yang bernama `Class Date`.
    
2. **Model Data Pengguna:** Saat seseorang mendaftar di web Anda, _Backend Developer_ akan mengambil data dari formulir dan langsung memasukkannya ke cetakan: `let penggunaBaru = new User(email, password)`. Ini memastikan setiap pengguna baru memiliki struktur data yang konsisten sebelum disimpan ke _database_.
    
3. **UI Components (Frontend):** Jika Anda memiliki daftar produk berisi 50 barang, Anda tidak menulis kode desain kotak HTML 50 kali. Anda membuat satu `Class ProductCard`, lalu menggunakan perulangan (_loop_) untuk mencetak 50 _object_ (kartu) tersebut ke layar, masing-masing dengan gambar dan harga yang berbeda.
