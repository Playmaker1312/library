## 1. Skenario Sistem (Apa yang akan kita bangun?)

Kita akan membuat sistem perangkat lunak untuk "Kafe Senja". Kafe ini menjual berbagai macam barang, tetapi secara garis besar terbagi menjadi dua: **Kopi** dan **Camilan**.

Kita ingin sistem ini bisa:

1. Menyimpan data nama menu, harga, dan **stok rahasia** (tidak boleh diubah sembarangan oleh kasir).
    
2. Memproses pesanan. Jika stok habis, pesanan ditolak.
    
3. Menyiapkan pesanan dengan cara yang berbeda: Kopi harus _diseduh_, sedangkan Camilan harus _dipanaskan_ (jika diminta pelanggan).
    

## 2. Peta Konsep OOP (Merakit 4 Pilar)

Sebelum menulis kode, seorang _Software Engineer_ selalu memetakan strateginya:

- **Abstraksi:** Kita buat _Class_ induk bernama `ItemMenu`. Ini adalah cetakan dasar. Kita tidak akan pernah menjual "ItemMenu" secara langsung (abstrak), kita hanya menjual bentuk nyatanya (Kopi/Camilan).
    
- **Enkapsulasi:** Atribut `stok` di dalam `ItemMenu` akan kita buat _Private_ (`#stok`). Kasir hanya bisa mengurangi stok lewat metode resmi (seperti `prosesPesanan()`).
    
- **Pewarisan (Inheritance):** Kita buat _Class_ `Kopi` dan `Camilan` yang mewarisi (_extends_) `ItemMenu` agar kita tidak perlu menulis ulang atribut nama dan harga.
    
- **Polimorfisme:** Kita buat metode `siapkanPesanan()`. `Kopi` akan menimpanya dengan perintah "Menyeduh air...", sedangkan `Camilan` akan menimpanya dengan perintah "Memanaskan di _microwave_...".
    

## 3. Implementasi Kode (JavaScript)

Mari kita tulis kodenya. Anda bisa menyalin kode ini dan menjalankannya di _browser console_ atau _Node.js_ untuk melihatnya hidup!

JavaScript

```
// ==========================================
// 1. ABSTRACT CLASS & ENCAPSULATION
// ==========================================
class ItemMenu {
    #stok; // Atribut Private (Hanya bisa diakses dari dalam sini)

    constructor(nama, harga, stokAwal) {
        this.nama = nama;
        this.harga = harga;
        this.#stok = stokAwal;
    }

    // Getter untuk mengecek stok dengan aman
    cekStok() {
        return this.#stok;
    }

    // Setter khusus dengan logika validasi
    kurangiStok() {
        if (this.#stok > 0) {
            this.#stok = this.#stok - 1;
            return true; // Berhasil dikurangi
        } else {
            return false; // Gagal, stok habis
        }
    }

    // Metode Abstrak (Wajib ditimpa oleh anak-anaknya)
    siapkanPesanan() {
        throw new Error("Metode ini wajib dibuat di Subclass!");
    }
}

// ==========================================
// 2. INHERITANCE (Pewarisan)
// ==========================================
class Kopi extends ItemMenu {
    constructor(nama, harga, stokAwal, jenisBiji) {
        super(nama, harga, stokAwal); // Memanggil constructor ItemMenu
        this.jenisBiji = jenisBiji;   // Atribut khusus Kopi
    }

    // 3. POLYMORPHISM (Method Overriding)
    siapkanPesanan() {
        console.log(`[BARISTA]: Menyeduh ${this.nama} menggunakan biji ${this.jenisBiji}... ☕`);
    }
}

class Camilan extends ItemMenu {
    constructor(nama, harga, stokAwal, perluDipanaskan) {
        super(nama, harga, stokAwal);
        this.perluDipanaskan = perluDipanaskan; // Boolean (true/false)
    }

    // 3. POLYMORPHISM (Method Overriding)
    siapkanPesanan() {
        if (this.perluDipanaskan) {
            console.log(`[DAPUR]: Memanaskan ${this.nama} di microwave... 🥐`);
        } else {
            console.log(`[DAPUR]: Mengambil ${this.nama} dari etalase... 🍰`);
        }
    }
}

// ==========================================
// 4. MENCETAK OBJECT & MENJALANKAN SISTEM
// ==========================================

// Kafe baru buka, kita siapkan menunya!
let menuEspresso = new Kopi("Espresso", 25000, 10, "Arabica");
let menuCroissant = new Camilan("Croissant Keju", 30000, 2, true);
let menuDonat = new Camilan("Donat Gula", 15000, 1, false);

// Fungsi Kasir untuk menerima pesanan pelanggan
function prosesPesanan(item) {
    console.log(`--- Ada pesanan: ${item.nama} ---`);
    
    // Cek apakah stok masih ada (Menggunakan metode resmi, bukan akses #stok langsung)
    if (item.kurangiStok()) {
        // Jika ada, siapkan! (Polimorfisme bekerja di sini, sistem tahu harus menyeduh/memanaskan)
        item.siapkanPesanan();
        console.log(`✅ Pesanan selesai! Total: Rp${item.harga}. Sisa stok: ${item.cekStok()}\n`);
    } else {
        console.log(`❌ Maaf, pesanan dibatalkan. Stok ${item.nama} sudah HABIS!\n`);
    }
}

// --- SIMULASI KASIR SEDANG BEKERJA ---
prosesPesanan(menuEspresso);   // Budi pesan Espresso
prosesPesanan(menuCroissant);  // Siti pesan Croissant
prosesPesanan(menuDonat);      // Andi pesan Donat
prosesPesanan(menuDonat);      // Joko pesan Donat JUGA (Tapi stoknya tadi cuma 1!)
```

## 4. Analisis Hasil Output

Jika Anda menjalankan sistem di atas, bagian pesanan Joko (pesanan terakhir) akan otomatis ditolak dan mencetak: `❌ Maaf, pesanan dibatalkan. Stok Donat Gula sudah HABIS!`.

Sistem ini sangat aman! Meskipun Joko adalah _hacker_ yang mencoba membobol kasir Anda dengan mengetik `menuDonat.#stok = 100`, program akan menolaknya karena atribut tersebut dilindungi oleh enkapsulasi.

## 5. Koneksi dengan Realitas Web Development

Jika ini adalah aplikasi React.js atau Node.js yang sebenarnya:

1. **Database:** Proses _instansiasi_ (`new Kopi(...)`) tidak dilakukan manual seperti di atas. Sistem akan menarik ratusan data dari _database_ (seperti MongoDB atau MySQL), melakukan _looping_, dan mencetak objek-objek tersebut secara dinamis.
    
2. **Frontend UI:** Fungsi `prosesPesanan()` tidak akan dijalankan lewat tulisan teks di konsol, melainkan dipicu (_triggered_) ketika kasir menekan `<button class="btn-pesan">` di layar sentuh iPad mereka.
    
3. **Modularitas:** _Class_ `ItemMenu`, `Kopi`, dan `Camilan` akan disimpan di file terpisah (seperti `menu.class.js`) lalu di-_import_ ke file utama aplikasi agar kodenya tidak menumpuk di satu tempat.
