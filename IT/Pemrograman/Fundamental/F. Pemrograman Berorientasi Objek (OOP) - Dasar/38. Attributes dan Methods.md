Mari kita bedah lebih dalam isi dari cetakan biru (_Class_) yang baru saja Anda buat. Sebuah objek yang baik tidak hanya diam seperti patung, ia memiliki identitas dan bisa melakukan sesuatu. Di sinilah dua komponen utama ini berperan.

---

## 1. Definisi Singkat

**Attributes (Atribut / Properti)** adalah variabel yang melekat pada sebuah objek. Atribut merepresentasikan **data, identitas, atau status (state)** dari objek tersebut (Kata Benda).

**Methods (Metode)** adalah fungsi yang melekat pada sebuah objek. Metode merepresentasikan **tindakan atau perilaku (behavior)** yang bisa dilakukan oleh objek tersebut, yang seringkali memanipulasi atribut di dalamnya (Kata Kerja).

## 2. Analogi: Televisi Pintar (Smart TV)

Bayangkan sebuah _Smart TV_ di ruang tamu Anda:

- **Atribut (Kondisi/Data saat ini):** TV Anda memiliki "Merk" (Samsung), "Ukuran" (55 Inch), "Status Daya" (Menyala), "Volume Saat Ini" (15), dan "Saluran Saat Ini" (Netflix). Ini adalah fakta-fakta pasif tentang TV tersebut pada detik ini.
    
- **Metode (Tindakan):** TV Anda memiliki tombol atau perintah untuk "Nyalakan/Matikan", "Tambah Volume", dan "Ganti Saluran".
    

Saat Anda menekan tombol "Tambah Volume" (memanggil _Method_), tindakan tersebut secara langsung akan mengubah angka "Volume Saat Ini" (memperbarui _Attribute_).

## 3. Poin-Poin Penting

- **Notasi Titik (Dot Notation):** Untuk mengakses atribut atau memanggil metode dari sebuah objek yang sudah dicetak, kita selalu menggunakan tanda titik (`.`). Misalnya: `tvSaya.volume` atau `tvSaya.tambahVolume()`.
    
- **Kata Kunci `this` (Sangat Krusial):** Di dalam sebuah _Class_, metode tidak bisa begitu saja memanggil nama atribut. Metode harus menggunakan kata ganti `this` (misal: `this.volume`). Ini untuk memberitahu komputer: _"Ubah volume milik objek ini saja, jangan ubah volume milik TV tetangga!"_
    
- **Pemisahan Tugas:** Atribut sebaiknya difokuskan murni untuk menyimpan nilai. Logika perhitungan, perulangan, atau percabangan (seperti mengecek apakah volume sudah maksimal) harus diletakkan di dalam Metode.
    

## 4. Contoh Implementasi

Mari kita terjemahkan analogi _Smart TV_ tadi ke dalam kode **JavaScript**:

JavaScript

```
class SmartTV {
    constructor(merkTV) {
        // --- ATRIBUT (Data / Status) ---
        this.merk = merkTV;
        this.isMenyala = false; // Default: TV mati saat baru dibeli
        this.volume = 10;       // Default: Volume awal 10
    }

    // --- METODE (Tindakan) ---
    // Metode 1: Mengubah status daya
    tekanTombolPower() {
        // Membalikkan keadaan: Jika false jadi true, jika true jadi false
        this.isMenyala = !this.isMenyala; 
        
        if (this.isMenyala) {
            console.log(`TV ${this.merk} MENYALA.`);
        } else {
            console.log(`TV ${this.merk} MATI.`);
        }
    }

    // Metode 2: Memanipulasi atribut volume dengan aturan logika
    tambahVolume() {
        if (this.isMenyala) {
            this.volume = this.volume + 1;
            console.log(`Volume naik menjadi: ${this.volume}`);
        } else {
            console.log("Tidak bisa tambah volume, TV masih mati!");
        }
    }
}

// Membaca (Instansiasi) Objek
let tvKamar = new SmartTV("LG");

// Memanggil Metode (Tindakan)
tvKamar.tambahVolume();      // Output: "Tidak bisa tambah volume, TV masih mati!"
tvKamar.tekanTombolPower();  // Output: "TV LG MENYALA."
tvKamar.tambahVolume();      // Output: "Volume naik menjadi: 11"

// Mengakses Atribut secara langsung
console.log(`Cek volume dari luar: ${tvKamar.volume}`); // Output: "Cek volume dari luar: 11"
```

## 5. Koneksi dengan Web Development

Pemahaman tentang _Attributes_ dan _Methods_ adalah kunci untuk menguasai interaksi di halaman web:

1. **Manipulasi Elemen HTML (DOM):** Setiap _tag_ HTML (`<div>`, `<img>`, `<button>`) yang dibaca oleh JavaScript akan diubah menjadi sebuah Objek raksasa.
    
    - Mengganti gambar? Anda mengubah atributnya: `gambarProfil.src = "foto-baru.jpg"`.
        
    - Menyembunyikan elemen? Anda mengubah atribut _style_: `kotakPesan.style.display = "none"`.
        
    - Memaksa sebuah input menjadi fokus kedip-kedip? Anda memanggil metodenya: `kolomEmail.focus()`.
        
2. **Validasi Formulir:** Saat Anda memiliki objek `UserRegistration`, Anda menyimpan `this.password` sebagai _Attribute_. Lalu Anda memiliki _Method_ `isValid()` yang bertugas mengecek apakah atribut _password_ tersebut panjangnya lebih dari 8 karakter.
    
3. **Animasi Game Web:** Jika Anda membuat game Mario Bros di web, karakter Mario adalah sebuah objek. Posisi koordinat X dan Y adalah _Attributes_. Tindakan melompat, menembak api, dan berjalan adalah _Methods_ yang mengubah koordinat tersebut pada setiap detiknya.
