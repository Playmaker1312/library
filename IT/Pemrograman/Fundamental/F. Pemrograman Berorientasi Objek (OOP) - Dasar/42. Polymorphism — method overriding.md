## 1. Definisi Singkat

**Polymorphism (Polimorfisme)** berasal dari bahasa Yunani: _Poly_ berarti "banyak" dan _Morph_ berarti "bentuk". Dalam pemrograman, ini adalah kemampuan berbagai objek dari _Class_ yang berbeda (namun masih satu keturunan) untuk merespons pemanggilan metode (_method_) yang nama aslinya persis sama, tetapi dengan cara atau perilaku yang berbeda-beda.

Senjata utama untuk mencapai bentuk ini adalah **Method Overriding**, di mana _Child Class_ menimpa metode milik _Parent Class_ dengan logikanya sendiri.

## 2. Analogi: Sistem Notifikasi Aplikasi

Bayangkan Anda adalah bos di sebuah perusahaan yang memiliki 3 jenis kurir penyampai pesan: Kurir Email, Kurir SMS, dan Kurir WhatsApp.

- Sebagai bos, Anda tidak mau pusing memikirkan cara kerja masing-masing kurir. Anda hanya ingin memberikan satu perintah baku: **"Kirim Pesan!"**
    
- **Polimorfisme** memungkinkan Anda membariskan ketiga kurir tersebut, lalu meneriakkan satu perintah yang sama: **"Kirim Pesan!"**.
    
- Kurir Email akan merespons perintah itu dengan menyalakan komputer dan mengirim email. Kurir SMS akan mengambil HP dan mengetik teks. Kurir WhatsApp akan membuka aplikasi WA.
    
- Satu perintah yang sama, direspons dengan banyak bentuk tindakan yang berbeda oleh objek yang berbeda.
    

## 3. Poin-Poin Penting

- **Satu Antarmuka, Banyak Implementasi:** Anda hanya perlu mengingat satu nama metode (misal: `kirim()`), tanpa perlu membuat metode dengan nama yang spesifik dan merepotkan seperti `kirimLewatEmail()`, `kirimLewatSMS()`, dst.
    
- **Skalabilitas Ekstrem:** Jika esok hari perusahaan Anda menambah "Kurir Telegram", Anda cukup membuat _Class_ baru yang menimpa metode `kirim()`. Kode utama bos yang meneriakkan perintah sama sekali tidak perlu diubah.
    
- **Kekuatan Array & Loop:** Sihir asli Polimorfisme baru benar-benar terasa ketika Anda mengumpulkan berbagai wujud objek ini ke dalam satu _Array_, lalu memanggil metode yang sama di dalam sebuah perulangan (_loop_).
    

## 4. Contoh Implementasi

Mari kita wujudkan analogi kurir notifikasi tadi menggunakan **JavaScript**:

JavaScript

```
// 1. SUPERCLASS (Parent)
class Notifikasi {
    kirim(pesan) {
        console.log(`Mengirim notifikasi sistem: ${pesan}`);
    }
}

// 2. SUBCLASS (Child 1)
class EmailNotifikasi extends Notifikasi {
    // METHOD OVERRIDING: Menimpa cara kerja 'kirim'
    kirim(pesan) {
        console.log(`[EMAIL]: Mengirim pesan "${pesan}" ke kotak masuk user.`);
    }
}

// 3. SUBCLASS (Child 2)
class SMSNotifikasi extends Notifikasi {
    // METHOD OVERRIDING: Menimpa cara kerja 'kirim'
    kirim(pesan) {
        console.log(`[SMS]: Mengirim teks "${pesan}" memotong pulsa user.`);
    }
}

// --- SIHIR POLIMORFISME BEKERJA ---

// Kita kumpulkan berbagai objek dengan "bentuk" berbeda ke dalam satu wadah
let daftarAlatNotif = [
    new EmailNotifikasi(), 
    new SMSNotifikasi(),
    new Notifikasi() // Parent asli juga ikut
];

// Kita putar loop, lalu panggil SATU perintah yang sama untuk semuanya!
daftarAlatNotif.forEach((alat) => {
    // Komputer secara cerdas tahu versi 'kirim' mana yang harus dijalankan
    alat.kirim("Promo Diskon 50% Hari Ini!"); 
});

/* OUTPUT:
[EMAIL]: Mengirim pesan "Promo Diskon 50% Hari Ini!" ke kotak masuk user.
[SMS]: Mengirim teks "Promo Diskon 50% Hari Ini!" memotong pulsa user.
Mengirim notifikasi sistem: Promo Diskon 50% Hari Ini!
*/
```

## 5. Koneksi dengan Web Development

Dalam dunia **Web Development**, polimorfisme adalah kunci untuk membangun sistem yang besar tanpa membuat _server_ Anda penuh dengan perintah `if-else` yang tiada henti:

1. **Sistem Pembayaran (Payment Gateway):** Saat pengguna memilih metode pembayaran di E-Commerce, _Backend Developer_ memiliki objek `KartuKredit`, `TransferBank`, dan `EWallet`. Daripada membuat ratusan kondisi `if`, _developer_ cukup memanggil perintah `metodeDipilih.prosesPembayaran()`. Setiap objek tahu cara memproses dirinya sendiri.
    
2. **Rendering UI (React/Vue):** Anda memiliki daftar data di _frontend_ yang berisi Gambar, Video, dan Teks. Anda membuat _class/component_ `VideoBlock`, `ImageBlock`, dan `TextBlock` yang semuanya memiliki metode `render()`. Sistem web cukup melakukan _looping_ dan memanggil `.render()` pada setiap blok tanpa peduli jenis medianya.
    
3. **Ekspor Data Beragam Format:** Aplikasi laporan keuangan Anda memiliki fitur "Download". Tombol ini memicu metode `cetak()`. Berkat polimorfisme, metode yang dieksekusi bisa saja `cetak()` milik PDF, `cetak()` milik Excel, atau `cetak()` milik CSV, tergantung objek apa yang aktif saat itu.
