## 1. Definisi Singkat

**Percabangan (Control Flow)** adalah struktur logika yang memungkinkan program untuk memilih blok kode mana yang akan dieksekusi berdasarkan suatu kondisi tertentu. Dengan `if`, `else if`, dan `else`, komputer tidak lagi menjalankan instruksi secara lurus dari atas ke bawah, melainkan bisa melompat atau mengabaikan instruksi tertentu layaknya manusia yang sedang membuat keputusan.

## 2. Analogi: Rambu Petunjuk Arah Jalan

Bayangkan Anda sedang mengemudi dan menemui persimpangan jalan dengan penjaga gerbang:

- **`if` (Kondisi Utama):** Ini adalah pertanyaan pertama penjaga. "Apakah tujuan Anda ke Jakarta?" Jika YA, gerbang kiri dibuka dan Anda masuk. Jika TIDAK, Anda lanjut ke pertanyaan berikutnya.
    
- **`else if` (Kondisi Alternatif):** "Oh bukan ke Jakarta, lalu apakah tujuan Anda ke Bandung?" Jika YA, gerbang tengah dibuka. Jika TIDAK, Anda lanjut lagi. (Anda bisa memiliki banyak penjaga `else if` berderet).
    
- **`else` (Sisa/Default):** Ini adalah jalan pembuangan terakhir. "Jika tujuan Anda BUKAN Jakarta dan BUKAN Bandung, maka silakan masuk ke gerbang kanan." Di sini penjaga tidak bertanya lagi, karena ini adalah satu-satunya jalan tersisa.
    

Begitu Anda masuk ke salah satu gerbang, Anda tidak akan bisa masuk ke gerbang lainnya. Program hanya memilih **satu** jalan.

## 3. Poin-Poin Penting

- **`if` (Wajib):** Setiap percabangan selalu dimulai dengan `if`. Di dalam tanda kurungnya, harus ada evaluasi yang menghasilkan `true` atau `false` (menggunakan operator perbandingan yang kita pelajari sebelumnya).
    
- **`else if` (Opsional):** Hanya akan dievaluasi oleh komputer jika `if` di atasnya bernilai `false`. Sangat berguna jika Anda memiliki lebih dari dua kemungkinan hasil.
    
- **`else` (Opsional tapi Penting):** Diletakkan di paling akhir penulisan tanpa kondisi kurung `()`. Ini berfungsi sebagai jaring pengaman (_fallback_) yang menangkap semua kondisi yang tidak sesuai dengan `if` maupun `else if`.
    
- **Saling Eksklusif:** Begitu komputer menemukan satu kondisi yang bernilai `true`, ia akan menjalankan kode di dalamnya dan **langsung mengabaikan** sisa pengecekan `else if` dan `else` di bawahnya.
    

## 4. Contoh Implementasi

Mari kita buat sistem pemberian _Grade_ nilai ujian menggunakan **JavaScript**. Ini adalah contoh nyata bagaimana program mengkategorikan data:

JavaScript

```
let nilaiSiswa = 85;

// Komputer mengecek dari atas ke bawah
if (nilaiSiswa >= 90) {
    // Jika nilai 90 ke atas
    console.log("Grade: A - Luar Biasa!");
} 
else if (nilaiSiswa >= 80) {
    // Jika nilai tidak sampai 90, tapi minimal 80
    console.log("Grade: B - Kerja Bagus!"); 
} 
else if (nilaiSiswa >= 70) {
    // Jika nilai tidak sampai 80, tapi minimal 70
    console.log("Grade: C - Cukup.");
} 
else {
    // Jika semua kondisi di atas salah (artinya nilai di bawah 70)
    console.log("Grade: D - Silakan ikut remedial.");
}

// Hasilnya untuk nilai 85 adalah: "Grade: B - Kerja Bagus!"
```

## 5. Koneksi dengan Web Development

Dalam **Web Development**, percabangan adalah jantung dari _User Experience_ (pengalaman pengguna) yang dinamis:

1. **Hak Akses (Role-Based Access):** Saat _user_ masuk ke aplikasi, program akan mengecek: `if (role === 'Admin')` tampilkan menu pengaturan karyawan, `else if (role === 'Manager')` tampilkan laporan keuangan, `else` tampilkan halaman pembeli biasa.
    
2. **Validasi Formulir:** Saat pengguna membuat _password_, percabangan digunakan untuk memberikan umpan balik: `if (password.length < 8)` munculkan teks merah "Password terlalu pendek!", `else` munculkan teks hijau "Password aman".
    
3. **Tampilan Responsif:** Meskipun CSS menangani tampilan utama, terkadang JavaScript butuh memanipulasi elemen berdasarkan ukuran layar: `if (lebarLayar < 768)` sembunyikan menu navigasi samping dan ubah menjadi tombol _hamburger_.
