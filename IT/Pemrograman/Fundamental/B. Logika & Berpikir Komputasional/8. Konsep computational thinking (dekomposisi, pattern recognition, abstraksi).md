## 1. Definisi Singkat

**_Computational Thinking_ (Berpikir Komputasional)** adalah proses berpikir sistematis untuk merumuskan masalah dan merancang solusinya sedemikian rupa sehingga solusi tersebut dapat dieksekusi secara efektif oleh komputer (atau manusia). Ini bukan tentang berpikir _seperti_ komputer, melainkan cara memecahkan masalah kompleks dengan pendekatan yang terstruktur.

## 2. Analogi: Menyelenggarakan Pesta Pernikahan

Bayangkan Anda ditugaskan mengurus pesta pernikahan untuk 1.000 tamu. Jika Anda melihatnya sebagai satu masalah raksasa ("Bagaimana cara bikin pesta?"), Anda akan panik. Anda harus menggunakan _Computational Thinking_:

- **Dekomposisi (Memecah Masalah):** Anda memecah "Pesta" menjadi bagian-bagian kecil: Katering, Gedung, Undangan, Pakaian, dan Hiburan. Mengurus katering jauh lebih mudah dipikirkan daripada mengurus seluruh pesta sekaligus.
    
- **Pengenalan Pola (Pattern Recognition):** Anda ingat bahwa pada pesta sebelumnya, setiap tamu rata-rata makan 2 porsi daging. Anda menggunakan _pola_ ini untuk memprediksi berapa banyak daging yang harus dipesan sekarang.
    
- **Abstraksi (Menyaring Informasi):** Saat memesan gedung, Anda fokus pada kapasitas (1.000 orang) dan harga. Anda _mengabaikan detail yang tidak penting_ pada tahap ini, seperti warna cat di toilet gedung atau merk lampu yang mereka gunakan.
    
- **Perancangan Algoritma (Langkah-langkah):** Anda membuat jadwal _rundown_ acara yang berurutan (1. Pembukaan, 2. Sambutan, 3. Makan, 4. Penutup) agar acara berjalan tanpa bentrok.
    

## 3. Poin-Poin Penting

Dalam dunia _Software Engineering_, _Computational Thinking_ terdiri dari 4 pilar utama:

- **Dekomposisi (_Decomposition_):** Menguraikan masalah sistem yang besar dan kompleks menjadi modul-modul, fungsi, atau komponen yang lebih kecil dan mudah dikelola/diuji.
    
- **Pengenalan Pola (_Pattern Recognition_):** Mengidentifikasi kesamaan data atau proses. Jika Anda menemukan pola berulang, Anda bisa menggunakan satu solusi (atau satu fungsi kode) untuk menyelesaikan banyak masalah serupa, sehingga menghindari duplikasi kerja.
    
- **Abstraksi (_Abstraction_):** Mengidentifikasi prinsip inti yang menghasilkan pola tersebut. Menyembunyikan kompleksitas teknis di latar belakang dan hanya memunculkan data yang benar-benar esensial untuk memecahkan masalah saat ini.
    
- **Algoritma (_Algorithm Design_):** Menyusun instruksi langkah demi langkah secara logis, sekuensial (berurutan), dan tidak ambigu untuk menyelesaikan masalah yang sudah dipecah dan diabstraksi tadi.
    

## 4. Contoh Implementasi

Misalkan Anda diminta membuat sistem **Login Pengguna**. Berikut cara berpikirnya:

1. **Dekomposisi:** Saya butuh (1) Tampilan form input, (2) Pengecekan data kosong, (3) Pencocokan dengan _database_.
    
2. **Pengenalan Pola:** Semua email pasti punya simbol `@` dan `.com/.id`. Saya bisa buat aturan (pola) untuk menolak input yang tidak punya simbol itu.
    
3. **Abstraksi:** Saat mencocokkan _user_, saya hanya peduli pada `email` dan `password`. Saya abaikan data `alamat` atau `foto profil` karena tidak relevan untuk proses login.
    
4. **Algoritma (diimplementasikan dalam JavaScript):**
    

JavaScript

```
// Langkah 1: Ambil data dari input pengguna
let emailInput = "budi@email.com";
let passwordInput = "rahasia123";

function prosesLogin(email, password) {
    // Langkah 2: Cek apakah input kosong (Pattern/Aturan)
    if (email === "" || password === "") {
        return "Email dan password harus diisi!";
    }
    
    // Langkah 3 & 4: Cek ke database (Abstraksi data esensial)
    if (email === "budi@email.com" && password === "rahasia123") {
        return "Login Berhasil! Selamat datang.";
    } else {
        return "Email atau password salah.";
    }
}

console.log(prosesLogin(emailInput, passwordInput));
```

## 5. Koneksi dengan Web Development

Dalam **Web Development**, _Computational Thinking_ adalah rutinitas sehari-hari:

1. **Membangun UI (User Interface):** Saat melihat desain halaman web raksasa (misal: Tokopedia atau Shopee), seorang _Frontend Developer_ akan melakukan **Dekomposisi**. Mereka tidak membuat halamannya sekaligus. Mereka memecahnya menjadi komponen kecil: _Header_, _Search Bar_, _Product Card_, dan _Footer_.
    
2. **Reusability (Penggunaan Ulang):** Dengan **Pengenalan Pola**, _developer_ menyadari bahwa ratusan produk di halaman depan memiliki desain kotak yang sama (hanya beda gambar dan harga). Jadi, mereka cukup menulis kode untuk satu _Product Card_, lalu menyuruh komputer mengulangnya ratusan kali (_looping_) dengan data yang berbeda.
    
3. **Efisiensi Sistem:** Menerapkan logika ini memastikan kode yang Anda tulis di JavaScript terstruktur dengan baik, mudah dibaca oleh _developer_ lain, dan tidak mudah _error_ saat fiturnya diperbesar di kemudian hari.
