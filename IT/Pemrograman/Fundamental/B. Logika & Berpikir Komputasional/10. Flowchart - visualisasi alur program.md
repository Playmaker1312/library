## 1. Definisi Singkat

**Flowchart (Diagram Alir)** adalah representasi visual dari sebuah algoritma atau proses program yang menggunakan bentuk-bentuk geometris standar yang dihubungkan dengan tanda panah. Ini adalah cara universal untuk memetakan langkah-langkah logika komputer secara grafis sebelum diubah menjadi baris kode.

## 2. Analogi: Peta Rute Perjalanan (GPS)

Bayangkan Anda sedang mencari alamat sebuah restoran:

- **Pseudocode** itu seperti instruksi teks: "Jalan lurus 100 meter, lalu belok kiri di perempatan, jika ada lampu merah berhenti, jika hijau jalan terus." Terkadang membacanya butuh konsentrasi ekstra.
    
- **Flowchart** itu seperti **tampilan layar Google Maps**. Anda melihat garis biru tebal yang menunjukkan arah (tanda panah), ikon penanda lokasi tujuan (titik akhir), dan ikon lampu lalu lintas (titik keputusan). Dengan satu lirikan mata, Anda sudah tahu ke mana arah perjalanan Anda dari awal hingga akhir.
    

## 3. Poin-Poin Penting

Dalam membuat flowchart untuk pemrograman, ada aturan bentuk (simbol) internasional yang wajib diikuti agar semua orang sepemahaman:

- **Oval (Terminal):** Menandakan titik **Mulai (Start)** dan **Selesai (End)** dari sebuah program.
    
- **Jajar Genjang (Input/Output):** Digunakan saat program menerima data (misal: _user_ mengetik _password_) atau menampilkan data (misal: _print_ "Halo di layar").
    
- **Persegi Panjang (Proses):** Melambangkan aksi atau perhitungan yang dilakukan komputer (misal: `total = harga - diskon`).
    
- **Belah Ketupat (Decision/Keputusan):** Titik persimpangan yang berisi pertanyaan "Ya atau Tidak" (misal: "Apakah _password_ benar?"). Dari sini, tanda panah akan bercabang ke dua arah yang berbeda.
    
- **Tanda Panah (Flowlines):** Menunjukkan arah aliran eksekusi program (biasanya dari atas ke bawah, atau kiri ke kanan).
    

## 4. Contoh Implementasi

Mari kita buat contoh kasus untuk fitur **Cek Kelulusan Ujian**.

**Bayangan Visual Flowchart-nya:**

1. **[Oval]** Mulai
    
2. **[Jajar Genjang]** Input nilai ujian
    
3. **[Belah Ketupat]** Apakah nilai >= 75?
    
    - _Jika YA (panah ke kanan)_ ➔ **[Jajar Genjang]** Tampilkan "Lulus!"
        
    - _Jika TIDAK (panah ke bawah)_ ➔ **[Jajar Genjang]** Tampilkan "Remedial"
        
4. **[Oval]** Selesai
    

**Implementasinya dalam JavaScript:** Flowchart di atas sangat mudah diterjemahkan ke dalam kode karena jalurnya sudah sangat jelas:

JavaScript

```
let nilaiUjian = 80; // Ini Input

// Ini adalah bagian Belah Ketupat (Keputusan)
if (nilaiUjian >= 75) {
    console.log("Lulus!"); // Panah 'YA'
} else {
    console.log("Remedial"); // Panah 'TIDAK'
}
// Selesai
```

## 5. Koneksi dengan Web Development

Dalam bidang **Web Development**, _flowchart_ adalah alat komunikasi utama antara tim bisnis, desainer, dan _programmer_:

1. **User Flow (Alur Pengguna):** Sebelum membuat halaman web, UI/UX Designer dan _Web Developer_ membuat _flowchart_ untuk perjalanan pengguna. Misalnya: "Buka _Landing Page_ ➔ Klik _Add to Cart_ ➔ Cek apakah sudah _Login_? (Belah Ketupat) ➔ Jika belum, arahkan ke halaman _Login_, jika sudah, arahkan ke halaman _Pembayaran_."
    
2. **Logika Backend yang Kompleks:** _Backend Developer_ wajib menggunakan _flowchart_ saat merancang fitur yang berhubungan dengan pihak ketiga, seperti _Payment Gateway_ (gerbang pembayaran bank). Alur pengecekan apakah pembayaran berhasil, gagal, atau tertunda harus dipetakan secara visual agar tidak ada uang pengguna yang "nyangkut" karena _error_ logika.
    
3. **Dokumentasi Proyek:** Saat _developer_ baru bergabung ke dalam tim, mereka tidak disuruh membaca ribuan baris kode, melainkan diberikan _flowchart_ agar langsung paham bagaimana struktur web tersebut bekerja.
