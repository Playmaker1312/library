Mari kita bahas salah satu konsep paling "mind-blowing" (mencengangkan) dalam pemrograman. Rekursi seringkali membuat _programmer_ pemula sakit kepala, tetapi begitu Anda memahaminya, Anda akan merasa memiliki kekuatan magis untuk memecahkan masalah yang sangat kompleks dengan kode yang sangat singkat.

---

## 1. Definisi Singkat

**Fungsi Rekursif** adalah sebuah fungsi yang **memanggil dirinya sendiri** dari dalam blok kodenya sendiri untuk menyelesaikan suatu masalah yang lebih kecil, hingga mencapai sebuah kondisi berhenti yang telah ditentukan secara absolut (_Base Case_). Sederhananya, ini adalah alternatif cara melakukan perulangan (_looping_) tanpa harus menggunakan perintah `for` atau `while`.

## 2. Analogi: Mencari Kunci di dalam Kotak Bersarang

Bayangkan Anda sedang mencari sebuah kunci, dan di depan Anda ada sebuah kotak besar. Saat kotak itu dibuka, ternyata di dalamnya ada kotak-kotak lain yang lebih kecil. Di dalam kotak kecil itu, ada kotak lagi. Bagaimana Anda menyelesaikannya?

- **Cara Biasa (For/While Loop):** Anda mengeluarkan semua kotak, menjeberkannya di lantai, lalu membukanya satu per satu dari ujung ke ujung.
    
- **Cara Rekursif:** Anda membuat satu instruksi baku bernama **"BukaKotak"**:
    
    1. Ambil satu kotak dan buka.
        
    2. **Jika** isinya adalah Kunci ➔ _Hore, selesai! Berhenti mencari._ (Ini disebut **Base Case**).
        
    3. **Jika** isinya adalah Kotak Baru ➔ _Lakukan instruksi **"BukaKotak"** pada kotak yang baru ditemukan ini._ (Ini disebut **Recursive Case**).
        

Anda mengulangi instruksi yang persis sama, namun pada ukuran masalah (kotak) yang semakin lama semakin kecil, hingga akhirnya menemukan tujuannya.

## 3. Poin-Poin Penting

Dalam merancang fungsi rekursif, ada dua komponen yang **wajib** ada. Jika salah satu hilang, program Anda akan meledak:

- **Base Case (Kondisi Berhenti):** Aturan kapan fungsi ini harus berhenti memanggil dirinya sendiri. Jika tidak ada _Base Case_, fungsi akan terjebak dalam perulangan tanpa henti (_Infinite Loop_).
    
- **Recursive Case (Kondisi Memanggil Ulang):** Bagian di mana fungsi memanggil dirinya sendiri, tetapi dengan data/argumen yang sudah dimodifikasi (biasanya nilainya dikurangi) agar semakin mendekati _Base Case_.
    
- **Hubungan dengan Stack (LIFO):** Ingat materi _Stack_ sebelumnya? Setiap kali fungsi memanggil dirinya sendiri, komputer menumpuk pemanggilan tersebut di memori (Call Stack). Saat _Base Case_ tercapai, tumpukan itu diselesaikan dan dikeluarkan satu per satu dari atas ke bawah.
    
- **Bahaya "Stack Overflow":** Jika Anda lupa menulis _Base Case_, tumpukan memori komputer akan penuh dan aplikasi Anda akan _crash_ secara instan. Pesan _error_-nya adalah _Stack Overflow_ (ya, dari sinilah nama forum tanya-jawab _programmer_ yang terkenal itu berasal!).
    

## 4. Contoh Implementasi

Contoh paling klasik untuk memahami rekursi adalah menghitung **Faktorial** dalam matematika (misal: Faktorial 5 atau `5!` adalah `5 * 4 * 3 * 2 * 1`).

Berikut adalah implementasinya di **JavaScript**:

JavaScript

```
// Deklarasi fungsi rekursif
function hitungFaktorial(angka) {
    
    // 1. BASE CASE (Kapan harus berhenti)
    // Jika angka sudah mencapai 1 atau 0, langsung kembalikan nilai 1
    if (angka === 1 || angka === 0) {
        return 1;
    }
    
    // 2. RECURSIVE CASE (Memanggil dirinya sendiri)
    // Kalikan angka saat ini dengan hasil faktorial dari (angka - 1)
    return angka * hitungFaktorial(angka - 1);
}

console.log("Hasil Faktorial 5 adalah: " + hitungFaktorial(5));
// Proses di belakang layar:
// 5 * hitungFaktorial(4)
// 5 * (4 * hitungFaktorial(3))
// 5 * (4 * (3 * hitungFaktorial(2)))
// 5 * (4 * (3 * (2 * hitungFaktorial(1))))
// 5 * (4 * (3 * (2 * 1)))  <-- Base Case tercapai, mulai dihitung!
// Hasil akhir: 120
```

## 5. Koneksi dengan Web Development

Anda mungkin bertanya, "Kapan saya butuh rekursi selain untuk matematika?" Dalam **Web Development**, rekursi adalah senjata rahasia untuk menangani data yang berlapis-lapis dan strukturnya tidak menentu (Struktur Pohon / _Tree_):

1. **Membaca Struktur HTML (DOM Tree):** Halaman web (_Document Object Model_) terdiri dari _tag_ HTML yang bersarang. Sebuah `<div>` bisa berisi `<ul>`, yang berisi `<li>`, yang berisi `<a>`. Jika Anda ingin menulis program untuk mencari teks tertentu di seluruh halaman, Anda tidak tahu berapa dalam lapisan HTML-nya. Anda menggunakan Fungsi Rekursif untuk "menggali" setiap _tag_ HTML sampai ke akar paling dalam.
    
2. **Membangun Menu Navigasi Dinamis:** Bayangkan Anda membuat menu untuk _e-commerce_ yang datanya dari _database_: _Elektronik -> Komputer -> Laptop -> Gaming -> Merk X_. Tingkat kedalaman kategorinya bisa sangat bervariasi. Fungsi rekursif sangat elegan untuk merender (_draw_) menu _dropdown_ bersarang seperti ini ke layar pengguna.
    
3. **Membaca Folder di Server (Node.js):** Saat _Backend Developer_ ingin menghapus sebuah direktori/folder di _server_ yang di dalamnya terdapat ratusan _sub-folder_ dan file, mereka menggunakan fungsi rekursif untuk masuk ke setiap _sub-folder_ dan menghapus isinya satu per satu.
