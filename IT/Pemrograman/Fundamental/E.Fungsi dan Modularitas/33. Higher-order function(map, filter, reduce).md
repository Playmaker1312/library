Mari kita bahas "Trio Suci" dalam manipulasi data modern. Di materi sebelumnya, Anda baru saja belajar tentang _Arrow Function_. Nah, ketiga fungsi yang akan kita bahas ini adalah alasan utama mengapa _Arrow Function_ diciptakan: untuk membuat manipulasi _Array_ menjadi sangat elegan dan singkat.

---

## 1. Definisi Singkat

**Higher-Order Function (Fungsi Tingkat Tinggi)** adalah fungsi yang dapat menerima fungsi lain sebagai argumen (parameter), atau mengembalikan fungsi lain sebagai hasil akhirnya. Tiga contoh paling populer yang wajib dikuasai setiap _programmer_ adalah **`map`**, **`filter`**, dan **`reduce`**. Ketiganya digunakan untuk mengolah data di dalam _Array_ tanpa merusak (mengubah) _Array_ aslinya (_Immutable_).

## 2. Analogi: Pabrik Pengolahan Kentang

Bayangkan Anda memiliki sekeranjang **Kentang Mentah** di sebuah ban berjalan pabrik:

- **`map` (Mengubah bentuk):** Mesin pemotong dan penggoreng. Setiap satu kentang mentah yang masuk akan diubah menjadi sebungkus _French Fries_. Jika ada 10 kentang mentah masuk, maka akan keluar persis 10 bungkus _French Fries_. _(Jumlah output selalu sama dengan input, tapi wujudnya berubah)._
    
- **`filter` (Menyaring):** Mesin _Quality Control_ (Kendali Mutu). Mesin ini mengecek: "Apakah kentang ini busuk?". Jika Ya, buang. Jika Tidak, loloskan. Jika dari 10 kentang ada 3 yang busuk, mesin ini hanya akan mengeluarkan 7 kentang bagus. _(Jumlah output bisa lebih sedikit dari input)._
    
- **`reduce` (Merangkum/Menggabungkan):** Mesin penumbuk (_Blender_). Anda memasukkan 10 kentang rebus yang terpisah-pisah, lalu mesin ini melumat dan menggabungkannya terus-menerus sampai akhirnya hanya keluar **satu** mangkuk besar _Mashed Potato_ (Kentang Tumbuk). _(Outputnya menyusut menjadi satu nilai akhir tunggal)._
    

## 3. Poin-Poin Penting

- **Tidak Merusak Data Asli (Non-Mutating):** Ketiga fungsi ini selalu menghasilkan data baru. _Array_ asli Anda tidak akan berubah, sehingga sangat aman dari _bug_ yang tidak disengaja.
    
- **Membutuhkan _Callback_:** Mereka tidak bisa bekerja sendiri. Anda harus menyuapkan sebuah _Anonymous/Arrow Function_ ke dalam kurungnya untuk memberitahu _apa_ aturan potongnya (`map`), _apa_ syarat lolosnya (`filter`), dan _bagaimana_ cara menumbuknya (`reduce`).
    
- **Sangat Deklaratif:** Anda hanya fokus pada **"Apa yang ingin dicapai"**, bukan **"Bagaimana cara mengulangnya"**. Anda tidak perlu lagi menulis inisialisasi `i = 0` atau memikirkan kapan `for loop` harus berhenti.
    

## 4. Contoh Implementasi

Mari kita lihat kekuatan ketiga mesin ini menggunakan **JavaScript**:

JavaScript

```
let daftarHarga = [10000, 25000, 50000, 15000];

// ==========================================
// 1. MAP: Mengubah setiap data
// Kasus: Kita ingin memberi diskon 10% untuk semua barang
// ==========================================
// Membaca: "Petakan setiap 'harga', kalikan dengan 0.9"
let hargaDiskon = daftarHarga.map((harga) => harga * 0.9);

console.log("Setelah Map:", hargaDiskon); 
// Output: [9000, 22500, 45000, 13500] (Jumlah tetap 4, nilainya berubah)


// ==========================================
// 2. FILTER: Menyaring data
// Kasus: Kita hanya ingin melihat barang yang harganya di atas 20000
// ==========================================
// Membaca: "Saring 'harga', loloskan HANYA JIKA harga > 20000"
let hargaMahal = daftarHarga.filter((harga) => harga > 20000);

console.log("Setelah Filter:", hargaMahal); 
// Output: [25000, 50000] (Tersisa 2 data yang memenuhi syarat)


// ==========================================
// 3. REDUCE: Merangkum data menjadi satu nilai
// Kasus: Menghitung TOTAL seluruh harga di keranjang belanja
// ==========================================
// Membaca: "Rangkum data. Ambil 'totalSementara', tambahkan dengan 'harga' saat ini. Mulai dari angka 0."
let totalBelanja = daftarHarga.reduce((totalSementara, harga) => totalSementara + harga, 0);

console.log("Setelah Reduce:", totalBelanja); 
// Output: 100000 (Semua angka digabung menjadi satu nilai bulat)

// Bukti bahwa Array asli tidak disentuh sama sekali:
console.log("Array Asli:", daftarHarga); // Output: [10000, 25000, 50000, 15000]
```

## 5. Koneksi dengan Web Development

Jika Anda berniat menjadi _Frontend Developer_ (terutama menggunakan React, Vue, atau Angular), `map`, `filter`, dan `reduce` adalah jantung dari tampilan web Anda:

1. **Rendering UI dengan `map`:** Di React.js, Anda tidak menulis tag HTML berulang-ulang. Anda mengambil Array berisi data dari _database_, lalu melakukan `map` untuk mencetak struktur HTML (seperti "Kartu Produk") sebanyak jumlah data tersebut.
    
2. **Sistem Pencarian dan Kategori dengan `filter`:** Saat pengguna mengetik "Sepatu Hitam" di kolom pencarian E-commerce, _developer_ menggunakan `filter` pada Array produk untuk membuang semua barang yang tidak mengandung kata "Hitam", lalu layar akan langsung memperbarui tampilannya secara instan.
    
3. **Halaman Checkout dengan `reduce`:** Saat melihat keranjang belanja (_Shopping Cart_), total harga, total berat untuk ongkos kirim, dan total pajak semuanya dihitung seketika menggunakan `reduce` berdasarkan item-item yang Anda masukkan.
