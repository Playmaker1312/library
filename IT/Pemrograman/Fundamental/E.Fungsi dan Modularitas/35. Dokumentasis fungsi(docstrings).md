## 1. Definisi Singkat

**Dokumentasi Fungsi (sering disebut _Docstrings_ atau _JSDoc_ di JavaScript)** adalah blok komentar terstruktur yang diletakkan tepat di atas sebuah fungsi (atau _class_). Tujuannya bukan sekadar menjelaskan _mengapa_ kode itu ditulis (seperti komentar biasa), melainkan mendeskripsikan secara resmi **apa** tugas fungsi tersebut, **data apa** yang harus dimasukkan (_parameters_), dan **apa hasil** yang akan dikeluarkan (_return value_).

## 2. Analogi: Label Informasi Gizi dan Buku Manual

Bayangkan Anda membeli obat flu di apotek:

- **Komentar Biasa (`//`)** itu seperti catatan di buku harian apoteker: _"Saya mencampur bahan kimia A dan B agar efek mengantuknya berkurang."_ (Hanya berguna bagi si peracik obat).
    
- **Docstrings / Dokumentasi Fungsi** itu seperti **Label Kemasan** di belakang botol obat: _"Fungsi: Meredakan gejala flu. Aturan Pakai (Parameter): Dewasa 2 sendok teh, Anak 1 sendok teh. Efek Samping (Return): Menyebabkan kantuk."_ Anda (sebagai pembeli) tidak perlu tahu rumus kimia obat tersebut, Anda hanya butuh membaca labelnya untuk bisa menggunakannya dengan aman.
    

## 3. Poin-Poin Penting

- **Berbeda dengan Komentar Biasa:** Komentar biasa diabaikan oleh alat pembaca kode. _Docstrings_ (seperti JSDoc) dibaca oleh _Code Editor_ Anda (seperti VS Code) untuk memberikan bantuan langsung.
    
- **Fitur Hover (IntelliSense):** Ini adalah sihir sesungguhnya. Jika Anda mendokumentasikan fungsi Anda dengan _Docstrings_, ketika _programmer_ lain memanggil fungsi Anda dan menempelkan _mouse_ (hover) di atas nama fungsinya, editor akan memunculkan _pop-up_ yang berisi buku manual dari fungsi tersebut!
    
- **Standar Industri:** Python menggunakan format `""" teks """` (disebut Docstrings), sedangkan JavaScript menggunakan `/** teks */` (disebut JSDoc).
    
- **Generator Web Otomatis:** Di perusahaan besar, _programmer_ tidak pernah membuat web dokumentasi secara manual. Alat khusus akan membaca semua _Docstrings_ di dalam kode, lalu menyulapnya menjadi _website_ dokumentasi yang cantik dan rapi secara otomatis.
    

## 4. Contoh Implementasi

Mari kita lihat bagaimana penulisan **JSDoc** di JavaScript. Perhatikan penggunaan dua bintang `/**` pada pembukaannya:

JavaScript

```
/**
 * Menghitung total harga setelah dikenakan diskon dan pajak.
 * Fungsi ini tidak akan mengubah harga asli di database.
 * * @param {number} hargaAsli - Harga awal barang sebelum diskon.
 * @param {number} persentaseDiskon - Diskon dalam bentuk persen (misal: 20 untuk 20%).
 * @returns {number} - Mengembalikan total harga bersih yang harus dibayar pembeli.
 */
function hitungTotalPembayaran(hargaAsli, persentaseDiskon) {
    let nilaiDiskon = hargaAsli * (persentaseDiskon / 100);
    let setelahDiskon = hargaAsli - nilaiDiskon;
    let pajak = setelahDiskon * 0.11; // PPN 11%
    
    return setelahDiskon + pajak;
}

// Saat Anda mengetik 'hitungTotalPembayaran(' di editor kode, 
// VS Code akan otomatis menampilkan kotak bantuan: 
// "hargaAsli (number): Harga awal barang sebelum diskon."
```

## 5. Koneksi dengan Web Development

Dalam dunia **Web Development** profesional, dokumentasi adalah alat komunikasi utama antar tim:

1. **Menggunakan NPM Packages (Library):** Saat Anda mengunduh alat buatan _developer_ luar dari internet (seperti _library_ pembuat grafik atau kalender), Anda tidak perlu membaca ratusan file kode mereka. Berkat _Docstrings_, Anda cukup mengetikkan fungsinya, dan _editor_ Anda akan memberitahu cara pakainya.
    
2. **Kerja Sama Frontend dan Backend:** _Backend Developer_ membuat API (jembatan data), lalu mendokumentasikannya menggunakan standar seperti _Swagger_ atau _OpenAPI_ (yang pada dasarnya adalah _Docstrings_ tingkat lanjut). _Frontend Developer_ tinggal membaca dokumen tersebut untuk mengetahui data apa saja yang harus mereka kirim dari formulir web.
    
3. **Onboarding Pegawai Baru:** Jika Anda masuk ke perusahaan rintisan (_startup_) dan diwarisi kode berusia 5 tahun, _Docstrings_ yang ditulis dengan baik akan menghemat waktu berbulan-bulan yang seharusnya dihabiskan untuk menebak-nebak apa fungsi dari `hitungTotal_v2()`.
