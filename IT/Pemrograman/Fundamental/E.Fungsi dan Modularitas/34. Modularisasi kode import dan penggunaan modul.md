Mari kita bahas bagaimana cara menjaga agar proyek pemrograman Anda tidak berubah menjadi benang kusut saat aplikasinya mulai membesar. Jika sebelumnya kita membungkus logika ke dalam fungsi, sekarang kita akan membungkus fungsi-fungsi tersebut ke dalam file-file yang terpisah.

---

## 1. Definisi Singkat

**Modularisasi Kode (Code Modularization)** adalah praktik memecah satu program besar menjadi file-file kecil yang mandiri dan terpisah, di mana setiap file (disebut **Modul**) memiliki tanggung jawab yang spesifik. Untuk menghubungkan file-file ini, kita menggunakan perintah **`export`** (menjadikan fungsi/variabel tersedia untuk digunakan di luar file) dan **`import`** (memanggil dan menggunakan fungsi/variabel dari file lain).

## 2. Analogi: Pabrik Perakitan Mobil

Bayangkan Anda sedang membangun sebuah pabrik mobil:

- Jika Anda mencoba membuat mesin, mengecat pintu, dan menjahit jok kulit di **satu meja kerja yang sama** (Satu file kode raksasa), pabrik Anda akan kacau balau, sesak, dan sangat sulit mencari sumber masalah jika mobilnya mogok.
    
- **Modularisasi** adalah memecah pabrik tersebut menjadi ruangan-ruangan spesifik:
    
    - _Ruang A_ khusus merakit mesin (Modul Mesin).
        
    - _Ruang B_ khusus menjahit jok (Modul Interior).
        
- Setelah mesin dan jok selesai, kepala pabrik di _Ruang Perakitan Utama_ (File Utama/Main) tinggal **mengimpor** (`import`) barang-barang jadi tersebut dari ruangan lain, lalu merakitnya menjadi satu mobil utuh.
    

## 3. Poin-Poin Penting

- **Separation of Concerns (Pemisahan Tugas):** Prinsip emas dalam pemrograman. Satu modul hanya boleh mengurus satu bidang spesifik. (Misal: `pembayaran.js` hanya mengurus uang, `autentikasi.js` hanya mengurus login password).
    
- **Mencegah "Spaghetti Code":** Mencari _bug_ di dalam 10 file yang masing-masing berisi 100 baris kode jauh lebih mudah dan waras daripada mencari _bug_ di dalam 1 file raksasa yang berisi 1.000 baris kode.
    
- **Reusability (Dapat Digunakan Ulang):** Jika Anda punya modul `kalkulatorDiskon.js`, Anda bisa mengimpornya ke halaman _Keranjang Belanja_, dan juga mengimpornya ke halaman _Admin Kasir_ tanpa perlu menulis ulang kodenya.
    
- **Namespace (Ruang Nama) Aman:** Variabel `total` di dalam `keranjang.js` tidak akan bertabrakan dengan variabel `total` di dalam `laporan.js` karena mereka berada di modul yang terisolasi, kecuali Anda secara eksplisit mengimpor keduanya bersamaan.
    

## 4. Contoh Implementasi

Dalam **JavaScript** modern (ES6 Modules), kita menggunakan _keyword_ `export` dan `import`. Mari kita lihat interaksi antara dua file yang berbeda:

**File 1: `matematika.js` (Modul yang menyediakan alat)**

JavaScript

```
// Kita membuat fungsi dan langsung menambahkan kata 'export' di depannya
export function tambah(a, b) {
    return a + b;
}

export function kali(a, b) {
    return a * b;
}

// Variabel juga bisa diekspor
export const phi = 3.14; 
```

**File 2: `app.js` (File utama yang menggunakan alat tersebut)**

JavaScript

```
// Kita MENGIMPOR fungsi dan variabel dari file matematika.js
// Perhatikan penggunaan kurung kurawal {} dan lokasi file './'
import { tambah, kali, phi } from './matematika.js';

let hargaBarang = tambah(50000, 20000);
console.log("Total Harga: " + hargaBarang); // Output: 70000

let luasLingkaran = kali(phi, 100);
console.log("Luas: " + luasLingkaran); // Output: 314
```

## 5. Koneksi dengan Web Development

Dalam **Web Development** modern, modularisasi bukan lagi sekadar pilihan, melainkan keharusan mutlak:

1. **Ekosistem NPM (Node Package Manager):** Ini adalah perpustakaan modul terbesar di dunia. Anda tidak perlu membuat fungsi konversi tanggal dari nol. Anda cukup mengunduh modul kalender buatan _developer_ lain dari internet, lalu melakukan `import moment from 'moment'` di proyek Anda.
    
2. **Komponen UI (React, Vue, Angular):** Saat Anda melihat _website_ seperti Netflix, tombol putar videonya mungkin adalah satu file modul bernama `PlayButton.jsx`, _navbar_ atasnya adalah modul `Navbar.jsx`, dan daftar filmnya adalah modul `MovieList.jsx`. Halaman utama hanyalah tempat untuk mengimpor dan menyusun komponen-komponen tersebut seperti balok Lego.
    
3. **Alat Bundling (Webpack / Vite):** Dulu, memuat 50 file JavaScript terpisah akan membuat _website_ sangat lambat karena _browser_ harus mengunduhnya satu per satu. Sekarang, kita menggunakan alat _Bundler_ (seperti Vite) di belakang layar. Alat ini akan membaca semua `import` Anda saat Anda merancang web, dan secara otomatis memadatkannya menjadi satu atau dua file kecil yang sangat cepat saat _website_ diluncurkan ke internet.
