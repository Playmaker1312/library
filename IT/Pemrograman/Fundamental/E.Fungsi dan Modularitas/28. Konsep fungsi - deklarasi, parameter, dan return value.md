## 1. Definisi Singkat

**Fungsi (Function)** adalah sebuah blok kode yang dirancang khusus untuk melakukan satu tugas tertentu dan dapat digunakan berulang kali (_reusable_). Fungsi menerima data masukan yang disebut **Parameter**, memprosesnya, dan memberikan hasil akhir yang disebut **Return Value** (Nilai Kembalian).

## 2. Analogi: Mesin Pembuat Jus (Blender)

Bayangkan sebuah fungsi itu seperti **Mesin Blender** di dapur Anda:

- **Deklarasi (Pembuatan):** Pabrik merakit mesin blender tersebut agar punya kemampuan memotong dan mengaduk. Mesin ini diam saja di dapur sampai Anda menyalakannya.
    
- **Parameter (Input):** Saat Anda ingin membuat jus, Anda memasukkan _Buah Apel_ dan _Susu_ ke dalam tabungnya. Apel dan Susu inilah parameternya (bahan baku yang bisa berubah-ubah, besok Anda bisa memasukkan Mangga).
    
- **Proses:** Anda menekan tombol "On". Mesin berputar menghancurkan bahan (kode di dalam fungsi sedang bekerja).
    
- **Return Value (Output):** Setelah selesai, Anda menuangkan hasilnya ke dalam gelas berupa _Jus Apel Susu_. Jika blender itu bocor atau tidak bisa dituang ke gelas (tidak ada _return value_), maka proses pembuatannya menjadi sia-sia karena Anda tidak bisa meminum hasilnya.
    

## 3. Poin-Poin Penting

- **Deklarasi vs Pemanggilan (Call/Invoke):** Menulis/mendeklarasikan fungsi tidak akan membuat kode di dalamnya berjalan. Anda harus "memanggil" nama fungsi tersebut agar kodenya dieksekusi.
    
- **Parameter vs Argumen:** _Parameter_ adalah nama variabel bayangan saat fungsi dibuat (misal: `buah`). _Argumen_ adalah data asli yang dikirim saat fungsi dipanggil (misal: `"Apel"`).
    
- **Keyword `return`:** Ini adalah pintu keluar dari fungsi. Begitu komputer membaca kata `return`, fungsi akan **langsung berhenti**, dan komputer akan membawa nilai di sebelah kata `return` itu kembali ke tempat fungsi tadi dipanggil.
    
- **Prinsip DRY (Don't Repeat Yourself):** Fungsi mencegah Anda menulis kode yang sama berkali-kali. Jika ada _bug_ pada cara perhitungan diskon, Anda cukup memperbaikinya di satu tempat (di dalam fungsi), bukan di 100 baris kode yang berbeda.
    

## 4. Contoh Implementasi

Mari kita buat fungsi untuk menghitung harga setelah diskon menggunakan **JavaScript**:

JavaScript

```
// 1. DEKLARASI FUNGSI dengan 2 PARAMETER (hargaAsli dan persentaseDiskon)
function hitungHargaAkhir(hargaAsli, persentaseDiskon) {
    // Proses perhitungan
    let potongan = hargaAsli * (persentaseDiskon / 100);
    let hargaAkhir = hargaAsli - potongan;
    
    // 2. RETURN VALUE (Mengirimkan hasil kembali ke luar fungsi)
    return hargaAkhir; 
    
    // Catatan: Kode apapun di bawah 'return' tidak akan pernah dijalankan!
    // console.log("Ini tidak akan tercetak"); 
}

// 3. PEMANGGILAN FUNGSI dengan ARGUMEN (100000 dan 20)
// Hasil dari fungsi (80000) ditangkap oleh variabel 'totalBayar'
let totalBayar = hitungHargaAkhir(100000, 20); 

console.log("Total yang harus dibayar: Rp" + totalBayar);

// Fungsi bisa dipanggil berulang kali dengan data berbeda!
let bayarSepatu = hitungHargaAkhir(500000, 50); 
console.log("Harga sepatu: Rp" + bayarSepatu);
```

## 5. Koneksi dengan Web Development

Dalam **Web Development**, 90% waktu Anda akan dihabiskan untuk memanggil atau menulis fungsi:

1. **Event Handler (Interaksi UI):** Saat Anda mengklik tombol "Kirim Pesan" di _website_, tombol tersebut sebenarnya memanggil sebuah fungsi di belakang layar (misalnya `function kirimData()`) yang bertugas mengambil teks Anda dan mengirimkannya ke _server_.
    
2. **Komponen UI (React/Vue):** Dalam _framework_ web modern seperti React, sebuah bagian tampilan web (misal: Kartu Profil User) aslinya adalah sebuah Fungsi! Parameter-nya adalah nama dan foto _user_, dan _Return Value_-nya adalah kode HTML untuk menampilkan kartu tersebut di layar.
    
3. **Memproses Data dari Database:** Saat _Backend Developer_ menerima data pengguna baru, mereka akan memasukkannya ke dalam fungsi `validasiEmail(emailUser)` untuk memastikan formatnya benar sebelum disimpan.
