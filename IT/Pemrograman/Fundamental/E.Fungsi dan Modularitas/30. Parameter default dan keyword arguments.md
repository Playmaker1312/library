## 1. Definisi Singkat

**Parameter Default** adalah nilai bawaan yang secara otomatis digunakan oleh fungsi jika pengguna (atau pemanggil fungsi) lupa atau tidak memberikan nilai pada parameter tersebut. **Keyword Arguments (Argumen Kata Kunci)** adalah cara memanggil fungsi dengan menyebutkan _nama_ parameternya secara eksplisit, sehingga Anda tidak perlu menghafal urutan posisinya.

## 2. Analogi: Memesan Kopi di Kafe

Bayangkan Anda sedang memesan minuman di sebuah kafe kekinian:

- **Parameter Default (Asumsi Standar):** Jika Anda hanya bilang, "Pesan Kopi Susu satu!", barista akan otomatis membuatkannya dengan **Gula Normal** dan **Es Normal**. _Gula Normal_ dan _Es Normal_ adalah _default_. Tapi, jika Anda bilang "Kopi Susu, gulanya sedikit saja," barista akan menimpa _default_ gula tersebut, namun tetap menggunakan _default_ es.
    
- **Keyword Arguments (Formulir Pesanan):** Tanpa _keyword arguments_, Anda harus menyebutkan pesanan sesuai urutan baku kasir: "(1) Kopi Susu, (2) Sedikit Gula, (3) Tanpa Es". Jika Anda terbalik menyebutkannya, kasir akan bingung. Dengan _keyword arguments_, Anda seperti mengisi formulir centang: `[Es: Tanpa Es]`, `[Minuman: Kopi Susu]`, `[Gula: Sedikit]`. Kasir paham pesanan Anda meskipun urutannya diacak-acak, karena setiap instruksi memiliki label (kata kunci) yang jelas.
    

## 3. Poin-Poin Penting

- **Mencegah Error (`undefined`):** Parameter default adalah jaring pengaman. Dalam JavaScript, jika argumen tidak diisi, nilainya menjadi `undefined`. Jika `undefined` ikut dihitung dalam operasi matematika, hasilnya akan menjadi `NaN` (Not a Number) dan merusak seluruh perhitungan aplikasi.
    
- **Penulisan Default:** Ditulis langsung di dalam tanda kurung saat fungsi dideklarasikan menggunakan tanda sama dengan (`=`). Contoh: `function sapa(nama = "Tamu")`.
    
- **Keyword Arguments di Berbagai Bahasa:** Bahasa seperti Python memiliki fitur ini secara bawaan (contoh: `buat_kopi(gula="sedikit", es="banyak")`). Di JavaScript, karena tidak ada fitur bawaan, _programmer_ menyiasatinya dengan teknik yang disebut **Object Destructuring** (mengirim argumen dalam bentuk _Dictionary/Object_).
    
- **Fleksibilitas Pemeliharaan:** Jika sebuah fungsi punya 5 parameter, mengingat urutan ke-5 parameter tersebut sangatlah menyiksa. Keyword arguments membebaskan Anda dari kewajiban menghafal urutan tersebut.
    

## 4. Contoh Implementasi

Mari kita lihat bagaimana kedua konsep ini diterapkan di **JavaScript**:

JavaScript

```
// ==========================================
// 1. PARAMETER DEFAULT
// ==========================================
// Jika 'nama' tidak diisi, otomatis pakai "Tamu"
// Jika 'bahasa' tidak diisi, otomatis pakai "ID"
function sapaPengguna(nama = "Tamu", bahasa = "ID") {
    if (bahasa === "ID") {
        console.log(`Halo, ${nama}!`);
    } else {
        console.log(`Hello, ${nama}!`);
    }
}

sapaPengguna("Budi", "EN"); // Output: "Hello, Budi!" (Menimpa semua)
sapaPengguna("Siti");       // Output: "Halo, Siti!" (Bahasa pakai default "ID")
sapaPengguna();             // Output: "Halo, Tamu!" (Semua pakai default)


// ==========================================
// 2. SIMULASI KEYWORD ARGUMENTS DI JAVASCRIPT
// ==========================================
// Menggunakan tanda kurung kurawal {} untuk menerima Object
function buatProfil({ nama, umur, kota = "Jakarta" }) {
    console.log(`${nama} berumur ${umur} tahun, tinggal di ${kota}.`);
}

// Saat memanggil fungsi, kita melempar sebuah Object.
// PERHATIKAN: Urutannya sengaja saya acak (umur dulu, baru nama)!
buatProfil({ 
    umur: 25, 
    nama: "Andi" 
    // Saya tidak mengirim 'kota', jadi otomatis pakai "Jakarta"
}); 
// Output: "Andi berumur 25 tahun, tinggal di Jakarta."
```

## 5. Koneksi dengan Web Development

Dalam **Web Development**, kedua konsep ini adalah standar industri mutlak untuk menulis kode yang bersih (_Clean Code_):

1. **Komponen UI (React.js):** Jika Anda merancang tombol (Button) kustom di React, Anda pasti menggunakan _Keyword Arguments_ (yang di React disebut _Props_) dan _Default Parameters_. Misalnya: `<Button warna="merah" teks="Hapus" />`. Anda tidak peduli urutan penulisan _warna_ atau _teks_, dan jika _warna_ tidak diisi, tombol akan otomatis berwarna biru (_default_).
    
2. **Konfigurasi Plugin/Library:** Saat Anda menggunakan _library_ pihak ketiga (seperti _slider_ gambar atau grafik _chart_), fungsi inisialisasinya bisa menerima puluhan opsi. _Library_ tersebut selalu menggunakan pola _Object/Keyword Arguments_ agar Anda cukup mengetikkan opsi yang ingin Anda ubah saja, sementara sisanya menggunakan nilai _default_ dari pembuat _library_.
    
3. **Memanggil API (Fetch):** Saat mengirim data ke _server_ menggunakan perintah `fetch()`, parameter keduanya adalah sebuah _Object_ opsi. Jika Anda hanya ingin mengambil data (_GET_), Anda tidak perlu mengisi apa-apa karena _default_-nya adalah _GET_. Tapi jika ingin mengirim data baru (_POST_), Anda cukup melampirkan _keyword_ `method: "POST"`.
