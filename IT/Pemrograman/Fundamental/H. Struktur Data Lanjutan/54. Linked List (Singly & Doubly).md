Selamat datang di dunia struktur data tingkat lanjut! Selama ini, setiap kali Anda ingin menyimpan kumpulan data, Anda selalu menggunakan _Array_. Namun, _Array_ memiliki kelemahan fatal dalam manajemen memori komputer.

Mari kita kenalan dengan **Linked List**, alternatif _Array_ yang jauh lebih fleksibel dan menjadi fondasi bagi struktur data kompleks lainnya.

---

## 1. Definisi Singkat

**Linked List (Senarai Berantai)** adalah struktur data linier di mana elemen-elemennya tidak disimpan di lokasi memori yang bersebelahan (tidak seperti _Array_).

Sebagai gantinya, setiap elemen disebut **Node (Simpul)**. Setiap _Node_ mengandung dua hal:

1. **Data:** Nilai atau informasi aktual yang ingin disimpan.
    
2. **Pointer (Penunjuk):** Alamat referensi yang menunjuk ke _Node_ berikutnya dalam urutan tersebut.
    

Titik awal dari Linked List disebut **Head (Kepala)**, dan titik akhirnya disebut **Tail (Ekor)** yang _pointer_-nya selalu menunjuk ke sesuatu yang kosong (`null`).

## 2. Analogi: Permainan Berburu Harta Karun

Bayangkan perbedaan antara _Array_ dan _Linked List_ seperti ini:

- **Array (Kompleks Perumahan):** Anda menyewa 5 rumah yang berderet persis bersebelahan (Blok A1 sampai A5). Jika Anda mencari rumah ke-3, Anda bisa langsung melompat ke sana dengan sangat cepat. Namun, jika Anda tiba-tiba butuh menyewa rumah ke-6 dan kebetulan rumah di sebelahnya (A6) sudah diisi orang lain, Anda harus memindahkan _seluruh_ barang Anda ke kompleks perumahan baru yang punya 6 rumah kosong berderet. Sangat tidak efisien!
    
- **Linked List (Berburu Harta Karun):** Anda meletakkan petunjuk di berbagai tempat yang acak (tersebar di memori komputer). Petunjuk pertama (_Head_) berisi selembar kertas: _"Pergi ke bawah pohon beringin"_. Di bawah pohon beringin (_Node_ 1), Anda menemukan hadiah (Data) dan selembar kertas lagi (_Pointer_): _"Pergi ke sumur tua"_.
    
    - Anda bisa menambah petunjuk baru kapan saja tanpa perlu memindahkan petunjuk yang lama. Cukup ubah saja tulisan di kertas petunjuk terakhir!
        

## 3. Singly vs Doubly Linked List

Ada dua varian utama dari struktur data ini, tergantung seberapa fleksibel Anda ingin menelusurinya:

### A. Singly Linked List (Satu Arah)

Ini adalah versi paling standar. Setiap _Node_ hanya tahu siapa "orang di depannya". Pergerakan hanya bisa dilakukan maju dari _Head_ ke _Tail_. Anda tidak bisa berjalan mundur.

### B. Doubly Linked List (Dua Arah)

Versi _upgrade_ yang lebih canggih. Selain memiliki _pointer_ `next` (menunjuk ke depan), setiap _Node_ juga memiliki _pointer_ `prev` (menunjuk ke belakang). Ini membuat Anda bisa berjalan maju dan mundur dengan bebas, meskipun menyedot sedikit lebih banyak memori untuk menyimpan alamat tambahan tersebut.

## 4. Poin-Poin Penting (Kapan Memakainya?)

- **Keunggulan Linked List (O(1) untuk Tambah/Hapus):** Menyisipkan atau menghapus data di tengah-tengah Linked List sangat cepat! Anda tidak perlu menggeser ribuan data lain (seperti di _Array_). Anda cukup memutus "tali penunjuk" dari satu _Node_ dan menyambungkannya ke _Node_ baru.
    
- **Kelemahan Linked List (O(n) untuk Pencarian):** Anda tidak bisa langsung mencari "Berikan saya elemen ke-100". Karena letaknya acak, Anda **wajib** berjalan dari _Head_, membaca petunjuk satu per satu sebanyak 100 kali. Pencarian sangat lambat dibandingkan _Array_.
    

## 5. Contoh Implementasi

Mari kita buat **Singly Linked List** sederhana menggunakan **JavaScript**. Kita akan menggunakan _Class_ OOP yang sudah Anda pelajari:

JavaScript

```
// 1. Membuat cetakan untuk satu Node (Satu kotak petunjuk)
class Node {
    constructor(data) {
        this.data = data; // Isinya
        this.next = null; // Penunjuk ke kotak selanjutnya (Awalnya kosong)
    }
}

// 2. Membuat cetakan untuk Rantai secara keseluruhan
class LinkedList {
    constructor() {
        this.head = null; // Titik awal (Awalnya belum ada apa-apa)
    }

    // Metode untuk menambahkan Node di akhir rantai
    tambahDiAkhir(data) {
        let nodeBaru = new Node(data);

        // Jika rantai masih kosong, jadikan node baru ini sebagai Head (Kepala)
        if (this.head === null) {
            this.head = nodeBaru;
            return;
        }

        // Jika sudah ada isinya, kita harus berjalan dari Head sampai ke Tail (ujung)
        let nodeSekarang = this.head;
        while (nodeSekarang.next !== null) {
            nodeSekarang = nodeSekarang.next; // Terus berjalan ke node berikutnya
        }

        // Setelah sampai di ujung, sambungkan tali ke node baru!
        nodeSekarang.next = nodeBaru;
    }

    // Metode untuk mencetak isi rantai
    cetakRantai() {
        let nodeSekarang = this.head;
        let hasil = "";
        while (nodeSekarang !== null) {
            hasil += nodeSekarang.data + " -> ";
            nodeSekarang = nodeSekarang.next;
        }
        console.log(hasil + "null");
    }
}

// --- MARI KITA UJI ---
let daftarTugas = new LinkedList();
daftarTugas.tambahDiAkhir("Bangun Tidur");
daftarTugas.tambahDiAkhir("Mandi");
daftarTugas.tambahDiAkhir("Ngoding");

daftarTugas.cetakRantai(); 
// Output: Bangun Tidur -> Mandi -> Ngoding -> null
```

## 6. Koneksi dengan Web Development

Meskipun Anda jarang menulis _Class Linked List_ dari nol di proyek web sehari-hari, konsepnya hidup di setiap sudut teknologi yang Anda gunakan:

1. **Tombol "Back" dan "Forward" di Browser:** Ini adalah implementasi murni dari **Doubly Linked List**. Halaman web saat ini adalah _Node_, tombol _Back_ melihat ke _pointer `prev`_, dan tombol _Forward_ melihat ke _pointer `next`_.
    
2. **Fitur Undo / Redo di Aplikasi:** Software seperti Microsoft Word atau VS Code menyimpan riwayat ketikan Anda dalam _Doubly Linked List_. Saat Anda menekan `Ctrl+Z` (Undo), ia mundur satu _Node_.
    
3. **React Fiber Architecture:** Jika Anda kelak mendalami React.js tingkat lanjut, mesin perenderan (rendering engine) terbaru mereka menggunakan struktur Linked List untuk menentukan komponen UI mana yang harus diperbarui lebih dulu dengan sangat efisien.
