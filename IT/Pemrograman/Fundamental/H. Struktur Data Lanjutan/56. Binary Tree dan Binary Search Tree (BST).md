Mari kita berkenalan dengan "selebriti" di dunia struktur data. Jika Anda mengikuti tes wawancara _coding_ di perusahaan teknologi raksasa, hampir dipastikan Anda akan bertemu dengan struktur yang satu ini.

Jika sebelumnya kita membahas _Tree_ secara umum (di mana satu _Parent_ bisa punya 10 atau 100 _Child_), sekarang kita akan membatasinya dengan aturan yang sangat ketat.

---

## 1. Definisi Singkat

- **Binary Tree (Pohon Biner):** Sebuah struktur pohon di mana setiap _Node_ (simpul) memiliki **maksimal dua anak** (disebut _Left Child_ / Anak Kiri dan _Right Child_ / Anak Kanan).
    
- **Binary Search Tree / BST (Pohon Pencarian Biner):** Ini adalah versi "pintar" dari Binary Tree. Ia memiliki satu aturan mutlak yang tidak boleh dilanggar:
    
    - Semua angka di cabang **KIRI** harus **LEBIH KECIL** dari induknya.
        
    - Semua angka di cabang **KANAN** harus **LEBIH BESAR** dari induknya.
        

## 2. Analogi: Permainan "Tebak Angka" yang Sempurna

Bayangkan Anda sedang bermain tebak angka dari 1 sampai 100 dengan teman Anda, dan teman Anda memikirkan angka **72**.

- **Binary Tree Biasa:** Seperti menaruh tebakan secara acak di cabang kiri atau kanan tanpa aturan. Anda tetap harus mengecek semuanya.
    
- **Binary Search Tree (BST):** Seperti strategi tebak angka yang mematikan.
    
    - _Root (Tebakan 1):_ Anda menebak **50**. Teman Anda bilang, "Lebih besar!".
        
    - Karena aturannya "yang lebih besar ada di kanan", Anda langsung **membuang seluruh cabang kiri** (angka 1-49).
        
    - _Node Kanan (Tebakan 2):_ Anda menebak **75**. Teman Anda bilang, "Lebih kecil!".
        
    - Anda belok ke cabang **kiri** dari 75 (membuang 76-100).
        
    - _Node Kiri (Tebakan 3):_ Anda menebak **62**. "Lebih besar!".
        
    - _Node Kanan (Tebakan 4):_ Anda menebak **72**. "Tepat!".
        

Hanya dalam 4 langkah, Anda menemukan 1 angka dari 100 kemungkinan!

## 3. Poin-Poin Penting

- **Kecepatan Logaritmik ($O(\log n)$):** Inilah alasan utama BST diciptakan. Setiap kali Anda melangkah turun ke satu cabang, Anda **mengeliminasi setengah** dari total sisa data. Mencari 1 data dari 1.000.000 data hanya butuh sekitar 20 langkah pengecekan!
    
- **Bisa Menjadi Bencana ($O(n)$):** Jika Anda memasukkan data yang _sudah berurutan_ (misal: 1, 2, 3, 4, 5) ke dalam BST, pohon tersebut akan tumbuh lurus terus ke kanan tanpa cabang kiri. Ini disebut _Skewed Tree_ (Pohon Condong). Kalau sudah begini, ia berubah menjadi _Linked List_ biasa dan pencariannya menjadi lambat ($O(n)$). (Ada algoritma lanjutan seperti _AVL Tree_ atau _Red-Black Tree_ untuk menyeimbangkan ini secara otomatis).
    
- **Dinamis:** Tidak seperti _Array_ yang ukurannya kaku, menambah atau menghapus data di dalam BST sangat cepat dan memori bisa melar sesuai kebutuhan.
    

## 4. Contoh Implementasi

Mari kita bangun kerangka dasar **Binary Search Tree** menggunakan **JavaScript**. Kita akan membuat _Node_ dan fungsi untuk memasukkan data (_Insert_) sesuai aturan BST.

JavaScript

```
// 1. Cetakan untuk satu Simpul (Node)
class Node {
    constructor(nilai) {
        this.nilai = nilai;
        this.kiri = null;  // Pointer ke anak lebih kecil
        this.kanan = null; // Pointer ke anak lebih besar
    }
}

// 2. Cetakan untuk Pohon (BST)
class BinarySearchTree {
    constructor() {
        this.root = null; // Akar pohon awalnya kosong
    }

    // Metode memasukkan data
    insert(nilai) {
        let nodeBaru = new Node(nilai);

        // Jika pohon kosong, ini jadi akar utamanya!
        if (this.root === null) {
            this.root = nodeBaru;
            return;
        }

        // Jika sudah ada akar, kita mulai pencarian letak yang pas
        let nodeSekarang = this.root;
        while (true) {
            if (nilai < nodeSekarang.nilai) {
                // Ke KIRI
                if (nodeSekarang.kiri === null) {
                    nodeSekarang.kiri = nodeBaru; // Taruh di sini!
                    return;
                }
                nodeSekarang = nodeSekarang.kiri; // Lanjut turun ke kiri
            } else {
                // Ke KANAN
                if (nodeSekarang.kanan === null) {
                    nodeSekarang.kanan = nodeBaru; // Taruh di sini!
                    return;
                }
                nodeSekarang = nodeSekarang.kanan; // Lanjut turun ke kanan
            }
        }
    }
}

// --- MARI KITA BANGUN POHONNYA ---
let pohonAngka = new BinarySearchTree();
pohonAngka.insert(50); // Ini jadi Root
pohonAngka.insert(30); // Lebih kecil, masuk KIRI
pohonAngka.insert(70); // Lebih besar, masuk KANAN
pohonAngka.insert(20); // Ke KIRI dari 50, lalu KIRI dari 30
pohonAngka.insert(40); // Ke KIRI dari 50, lalu KANAN dari 30

console.log(pohonAngka.root.kiri.kanan.nilai); // Output: 40
```

## 5. Koneksi dengan Web Development

Pemahaman tentang BST sangat krusial ketika aplikasi web Anda mulai menangani data skala besar:

1. **Database Indexing (B-Trees):** Saat Anda menggunakan PostgreSQL atau MySQL, mesin _database_ tersebut tidak menggunakan _Array_ untuk mencari data. Mereka menggunakan variasi tingkat tinggi dari BST (yang disebut _B-Tree_) agar _query_ pencarian data produk atau _user_ bisa selesai dalam hitungan milidetik.
    
2. **Mesin JavaScript (V8 Engine):** Saat Anda menggunakan struktur data `Set` atau `Map` di JavaScript modern untuk menyimpan data unik, di belakang layar mesin C++ dari _browser_ Anda seringkali menggunakan variasi _Binary Tree_ untuk mengelolanya.
    
3. **3D Rendering (Game Web/WebGL):** Jika Anda membuat aplikasi 3D di _browser_, _engine_ grafis menggunakan _BSP Tree_ (Binary Space Partitioning) untuk dengan cepat menentukan objek 3D mana yang tertutup oleh objek lain, sehingga _browser_ tidak perlu membuang tenaga menggambar objek yang tidak terlihat oleh kamera.
