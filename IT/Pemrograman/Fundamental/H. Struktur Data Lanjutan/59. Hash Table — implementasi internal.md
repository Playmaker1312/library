Mari kita bongkar salah satu "sihir" paling kuat di dunia pemrograman. Selama ini, Anda sangat sering menggunakan _Object_ di JavaScript atau _Dictionary_ di Python untuk mencari data secara instan. Pernahkah Anda bertanya-tanya, bagaimana cara komputer bisa menemukan data secepat itu tanpa melakukan perulangan (_looping_)?

Rahasia di balik kecepatan kilat tersebut adalah **Hash Table**.

---

## 1. Definisi Singkat

**Hash Table (Tabel Hash)** adalah struktur data yang memetakan sebuah kunci (_Key_) ke sebuah nilai (_Value_) demi mencapai kecepatan pencarian, penambahan, dan penghapusan data dalam waktu konstan atau **O(1)**.

Secara internal, _Hash Table_ sebenarnya hanyalah sebuah **Array biasa**. Rahasia utamanya terletak pada sebuah fungsi matematis khusus yang disebut **Hash Function (Fungsi Hash)**, yang bertugas mengubah teks kunci (misal: "Budi") menjadi angka indeks _Array_ (misal: indeks ke-4).

## 2. Analogi: Pustakawan Magis

Bayangkan sebuah perpustakaan raksasa dengan jutaan buku (Array), dan Anda ingin mencari buku berjudul "Harry Potter".

- **Cara Biasa (Array/List):** Anda berjalan dari rak nomor 1, mengecek setiap buku satu per satu. Sangat lambat $O(n)$.
    
- **Cara Hash Table:** Anda mendatangi seorang Pustakawan Magis (_Hash Function_). Anda menyebutkan judul "Harry Potter" (_Key_). Pustakawan ini memiliki rumus matematika di kepalanya yang langsung menerjemahkan huruf-huruf di judul tersebut menjadi sebuah angka pasti. Ia langsung menjawab: _"Rak nomor 842!"_. Anda berjalan lurus ke rak 842 dan langsung menemukan bukunya. Sangat cepat $O(1)$!
    

## 3. Poin-Poin Penting (Anatomi Internal)

Untuk membangun _Hash Table_ dari nol, ada tiga komponen internal yang harus ditangani:

- **Hash Function:** Fungsi ini menerima _string_ dan mengembalikan angka integer. Fungsi ini harus konsisten (kata "Apel" harus selalu menghasilkan angka yang sama) dan harus mendistribusikan angka secara merata. Rumus dasarnya sering kali melibatkan operasi modulo agar indeks tidak melebihi batas kapasitas _Array_: $h(k) = \text{hash}(k) \bmod m$ (di mana $m$ adalah ukuran _Array_).
    
- **Buckets (Ember Penampung):** Ini adalah slot-slot _Array_ di bawah kap mesin yang digunakan untuk menyimpan nilai sebenarnya.
    
- **Collision (Tabrakan):** Apa yang terjadi jika kata "Apel" dan "Jeruk" ternyata menghasilkan angka indeks yang sama oleh _Hash Function_? Ini disebut _Collision_. Solusi paling umum adalah **Chaining**: Alih-alih menyimpan satu nilai, slot indeks tersebut diubah menjadi sebuah _Linked List_ atau _Array_ kecil untuk menampung kedua nilai tersebut secara berderet.
    

## 4. Contoh Implementasi

Mari kita bangun _Hash Table_ kita sendiri dari nol menggunakan **JavaScript** agar Anda bisa melihat mekanisme _Hash Function_ dan _Array_ internalnya:

JavaScript

```
class HashTableCustom {
    constructor(ukuran) {
        // Kita siapkan Array kosong dengan ukuran tertentu sebagai "Rak"
        this.rak = new Array(ukuran);
    }

    // 1. FUNGSI HASH (Pustakawan Magis)
    // Mengubah string (kunci) menjadi angka indeks (0 sampai ukuran rak)
    _hash(kunci) {
        let total = 0;
        for (let i = 0; i < kunci.length; i++) {
            // Mengambil nilai angka dari karakter ASCII
            total += kunci.charCodeAt(i); 
        }
        // Pastikan angkanya tidak melebihi jumlah rak kita
        return total % this.rak.length; 
    }

    // 2. METODE SET (Menyimpan Data)
    set(kunci, nilai) {
        let indeks = this._hash(kunci);
        
        // Jika rak di indeks ini masih kosong, buat wadah (Array) baru
        if (!this.rak[indeks]) {
            this.rak[indeks] = [];
        }
        
        // Simpan pasangan [kunci, nilai] ke dalam rak tersebut (Solusi Tabrakan / Chaining)
        this.rak[indeks].push([kunci, nilai]);
        console.log(`Menyimpan '${kunci}' di indeks: ${indeks}`);
    }

    // 3. METODE GET (Mencari Data dengan kecepatan O(1))
    get(kunci) {
        let indeks = this._hash(kunci);
        let wadah = this.rak[indeks];

        if (wadah) {
            // Jika ada tabrakan, kita loop sebentar di dalam wadah kecil ini
            for (let i = 0; i < wadah.length; i++) {
                if (wadah[i][0] === kunci) {
                    return wadah[i][1]; // Kembalikan nilainya
                }
            }
        }
        return undefined; // Data tidak ditemukan
    }
}

// --- MARI KITA UJI ---
let memoriCepat = new HashTableCustom(50); // Buat 50 rak kosong

memoriCepat.set("apel", 15000);  // Output: Menyimpan 'apel' di indeks: ...
memoriCepat.set("jeruk", 20000); // Output: Menyimpan 'jeruk' di indeks: ...

// Mengambil data secara instan tanpa perlu meloop 50 rak!
console.log("Harga Apel: Rp" + memoriCepat.get("apel")); // Output: Harga Apel: Rp15000
```

## 5. Koneksi dengan Web Development

_Hash Table_ adalah struktur data pendukung infrastruktur internet modern:

1. **Sistem Caching (Redis/Memcached):** Saat _Backend Developer_ menyimpan data sementara yang sering diakses (seperti sesi _login user_ atau keranjang belanja), mereka menggunakan _database in-memory_ seperti Redis yang murni beroperasi di atas konsep _Hash Table_ agar pengambilannya super cepat.
    
2. **JavaScript Objects & Sets:** Setiap kali Anda membuat `let user = { nama: "Budi" }`, JavaScript _engine_ di belakang layar sering kali mengonversinya menjadi _Hash Table_ untuk memastikan Anda bisa mengakses `user.nama` secara instan.
    
3. **Kriptografi & Keamanan Kata Sandi:** Meskipun konsepnya berbeda (Fungsi Hash Kriptografi vs Fungsi Hash Data), algoritma seperti SHA-256 yang mengubah kata sandi pengguna menjadi teks acak tidak bisa dikembalikan terinspirasi dari prinsip deterministik _Hash Function_ ini.
