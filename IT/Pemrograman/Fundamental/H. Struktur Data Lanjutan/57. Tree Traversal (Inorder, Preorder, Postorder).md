Mari kita pecahkan sebuah masalah logis. Di _Array_, membaca data sangatlah mudah: Anda tinggal membuat perulangan dari indeks `0` sampai akhir. Namun, bagaimana cara Anda "membaca" data di dalam sebuah Pohon (_Tree_) yang memiliki banyak cabang dan tidak berbaris lurus?

Di sinilah kita membutuhkan teknik navigasi khusus yang disebut **Tree Traversal** (Penelusuran Pohon).

---

## 1. Definisi Singkat

**Tree Traversal** adalah proses mengunjungi setiap _Node_ (simpul) di dalam sebuah pohon tepat satu kali. Karena struktur pohon tidak linier (bercabang), kita tidak bisa membacanya dari kiri ke kanan begitu saja.

Dalam penelusuran berkedalaman (_Depth-First Search_), terdapat tiga rute standar yang paling sering digunakan, yang dinamai berdasarkan kapan kita mengunjungi **Root (Akar/Induk)**:

1. **Inorder:** Kiri ➔ Akar ➔ Kanan
    
2. **Preorder:** Akar ➔ Kiri ➔ Kanan
    
3. **Postorder:** Kiri ➔ Kanan ➔ Akar
    

## 2. Analogi: Komandan dan Dua Pasukan

Bayangkan sebuah peleton tentara. Setiap peleton memiliki satu **Komandan (Root)**, **Pasukan Sayap Kiri (Left)**, dan **Pasukan Sayap Kanan (Right)**. Mereka harus memberikan laporan.

- **Inorder (Kiri - Komandan - Kanan):** Pasukan Kiri melapor lebih dulu, disusul Komandan, dan ditutup oleh Pasukan Kanan. (Sangat rapi dan berurutan).
    
- **Preorder (Komandan - Kiri - Kanan):** Komandan melapor paling pertama (Pre = Awal), lalu menyuruh Pasukan Kiri, dan terakhir Pasukan Kanan. (Cocok untuk menyalin struktur komando dari atas ke bawah).
    
- **Postorder (Kiri - Kanan - Komandan):** Pasukan Kiri dan Kanan disuruh maju bertempur (melapor) lebih dulu. Komandan adalah orang terakhir yang melapor setelah semua bawahannya selesai (Post = Akhir).
    

## 3. Poin-Poin Penting dan Kegunaannya

Ketiga metode ini bukan sekadar variasi tanpa tujuan. Masing-masing memiliki "kekuatan super" tersendiri:

- **Sihir Inorder (Mengurutkan Data):** Jika Anda menggunakan metode _Inorder_ pada sebuah _Binary Search Tree_ (BST), angka-angka yang asalnya bercabang secara hierarkis akan **tercetak secara otomatis dari nilai terkecil hingga terbesar**. Ini adalah cara termudah mengurutkan data di dalam pohon!
    
- **Sihir Preorder (Menduplikasi Pohon):** Jika Anda ingin menyalin atau membuat _clone_ dari sebuah pohon ke komputer lain, Anda harus menggunakan _Preorder_. Anda harus membuat _Root_-nya (induknya) terlebih dahulu sebelum bisa menempelkan anak-anaknya ke bawah.
    
- **Sihir Postorder (Menghapus Pohon):** Jika Anda ingin menghancurkan/menghapus sebuah pohon dari memori, Anda harus menggunakan _Postorder_. Anda tidak boleh membunuh induknya sebelum memastikan semua anak-anaknya sudah dihapus lebih dulu (jika induknya dihapus duluan, anak-anaknya akan menjadi data "yatim piatu" yang memenuhi memori komputer).
    

## 4. Contoh Implementasi

Karena struktur pohon selalu berulang secara pola (sebuah cabang memiliki cabang lagi), implementasi terbaik untuk penelusuran ini adalah menggunakan **Fungsi Rekursif**.

Berikut adalah contoh kodenya di **JavaScript**:

JavaScript

```
// Menggunakan struktur Node dari materi BST sebelumnya
// Node memiliki atribut: nilai, kiri, dan kanan

// 1. INORDER (Kiri -> Akar -> Kanan)
// Mengurutkan data dari terkecil ke terbesar
function cetakInorder(node) {
    if (node !== null) {
        cetakInorder(node.kiri);       // 1. Kunjungi seluruh cabang kiri
        console.log(node.nilai);       // 2. Cetak nilai saat ini (Akar)
        cetakInorder(node.kanan);      // 3. Kunjungi seluruh cabang kanan
    }
}

// 2. PREORDER (Akar -> Kiri -> Kanan)
// Mencetak struktur dari atas ke bawah
function cetakPreorder(node) {
    if (node !== null) {
        console.log(node.nilai);       // 1. Cetak nilai saat ini (Akar)
        cetakPreorder(node.kiri);      // 2. Kunjungi seluruh cabang kiri
        cetakPreorder(node.kanan);     // 3. Kunjungi seluruh cabang kanan
    }
}

// 3. POSTORDER (Kiri -> Kanan -> Akar)
// Memproses dari daun terbawah hingga ke atas
function cetakPostorder(node) {
    if (node !== null) {
        cetakPostorder(node.kiri);     // 1. Kunjungi seluruh cabang kiri
        cetakPostorder(node.kanan);    // 2. Kunjungi seluruh cabang kanan
        console.log(node.nilai);       // 3. Cetak nilai saat ini (Akar)
    }
}
```

_Perhatikan bahwa ketiga fungsi di atas kode dasarnya sama persis, Anda HANYA memindahkan baris `console.log(node.nilai)` ke atas, tengah, atau bawah!_

## 5. Koneksi dengan Web Development

Penelusuran hierarki ini adalah fondasi dari banyak alat yang Anda gunakan sebagai _developer_:

1. **Menghapus Folder (Node.js):** Saat _Backend Developer_ menggunakan perintah untuk menghapus folder proyek yang berisi ratusan _sub-folder_ dan file, sistem operasi menggunakan algoritma **Postorder**. Ia menghapus file di tingkat terdalam dulu, baru menghapus folder induknya.
    
2. **Manipulasi DOM (Frontend):** Saat Anda menggunakan `document.querySelectorAll()`, _browser_ sering kali menggunakan variasi penelusuran pohon (seperti _Preorder_) untuk membaca struktur HTML dari tag `<html>` teratas turun ke `<p>` atau `<a>` di bawahnya untuk mencari elemen yang cocok.
    
3. **Compiler Code (Babel/Webpack):** Alat pengemas kode web modern mengubah kode JavaScript Anda menjadi sebuah pohon yang disebut _Abstract Syntax Tree_ (AST). Alat ini lalu melakukan _Inorder_ atau _Postorder traversal_ pada pohon tersebut untuk mengubah sintaks modern menjadi kode jadul yang bisa dibaca _browser_ lama.
