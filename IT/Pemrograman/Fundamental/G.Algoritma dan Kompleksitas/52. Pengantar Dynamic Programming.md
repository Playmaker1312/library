Mari kita bahas salah satu teknik optimasi paling brilian yang pernah diciptakan dalam ilmu komputer. Jika _Divide and Conquer_ (di materi sebelumnya) mengajarkan kita untuk memecah masalah, maka **Dynamic Programming** mengajarkan kita untuk memecah masalah **dan memiliki ingatan yang tajam** agar tidak melakukan pekerjaan bodoh yang berulang-ulang.

---

## 1. Definisi Singkat

**Dynamic Programming (Pemrograman Dinamis)** adalah sebuah teknik optimasi algoritma yang digunakan ketika sebuah masalah besar bisa dipecah menjadi sub-masalah yang lebih kecil, **tetapi sub-masalah tersebut saling tumpang tindih (diulang-ulang)**.

Alih-alih memecahkan sub-masalah yang sama berkali-kali secara mubazir, kita memecahkannya **satu kali saja**, lalu menyimpan hasilnya di dalam sebuah "buku catatan" (memori/cache). Jika di kemudian hari komputer disuruh memecahkan masalah yang sama, ia tinggal mencontek dari buku catatan tersebut. Teknik mencatat ini disebut **Memoization (Memoisasi)**.

## 2. Analogi: Menghitung Angka di Papan Tulis

Bayangkan saya menulis penjumlahan panjang di papan tulis:

**1 + 1 + 1 + 1 + 1 + 1 + 1 + 1 = ?**

Anda menghitungnya secara manual dan menjawab: **"8!"**

Lalu, di sebelah angka 8 tersebut, saya menambahkan satu angka lagi:

**1 + 1 + 1 + 1 + 1 + 1 + 1 + 1 + 1 = ?**

Apakah Anda akan menghitung ulang dari angka 1 yang pertama di ujung kiri? Tentu saja tidak! Anda langsung menjawab **"9!"** karena Anda **ingat** bahwa deretan sebelumnya berjumlah 8, dan Anda tinggal menambahkan 1.

Itulah inti dari Dynamic Programming: **Mengingat masa lalu untuk mempercepat masa depan.**

## 3. Poin-Poin Penting

Agar sebuah masalah bisa diselesaikan dengan Dynamic Programming, ia harus memiliki dua sifat ini:

- **Overlapping Subproblems (Masalah Tumpang Tindih):** Masalah tersebut harus menuntut komputer memanggil fungsi dengan parameter yang sama berkali-kali. (Inilah mengapa _Merge Sort_ tidak butuh DP, karena _Merge Sort_ selalu membelah data yang baru dan berbeda).
    
- **Memoization (Top-Down):** Pendekatan di mana kita menggunakan fungsi rekursif yang bergerak dari masalah terbesar ke terkecil, namun kita menyiapkan objek/variabel _cache_ untuk menyimpan setiap hasil yang baru pertama kali dihitung.
    
- **Tabulation (Bottom-Up):** Alternatif dari Memoisasi. Pendekatan di mana kita tidak menggunakan fungsi rekursif, melainkan menggunakan perulangan biasa (seperti _for loop_) untuk mengisi sebuah Array/Tabel dari nilai terkecil hingga mencapai nilai akhir yang dicari.
    

## 4. Contoh Implementasi

Masalah paling klasik yang membuktikan kehebatan DP adalah menghitung **Deret Fibonacci** (di mana angka saat ini adalah jumlah dari dua angka sebelumnya: 0, 1, 1, 2, 3, 5, 8, 13...).

Mari kita lihat perbandingan fungsi rekursif biasa vs fungsi dengan **Memoization** di JavaScript:

JavaScript

```
// ==========================================
// CARA LAMA (Rekursif Biasa) - SANGAT LAMBAT: O(2^n)
// ==========================================
function fibonacciLama(n) {
    if (n <= 1) return n;
    // Komputer menghitung hal yang SAMA berkali-kali secara membabi buta!
    return fibonacciLama(n - 1) + fibonacciLama(n - 2); 
}

// Jika Anda menjalankan fibonacciLama(50), komputer Anda akan hang 
// karena ia harus memanggil fungsi ini bertriliun-triliun kali!


// ==========================================
// CARA MODERN (Dynamic Programming - Memoization) - SUPER CEPAT: O(n)
// ==========================================
// Kita siapkan "buku catatan" (cache) berbentuk Object
function fibonacciCerdas(n, cache = {}) {
    // 1. Cek buku catatan. Kalau angkanya sudah pernah dihitung, langsung contek hasilnya!
    if (n in cache) {
        return cache[n];
    }
    
    // 2. Base case
    if (n <= 1) return n;
    
    // 3. Kalau belum ada di catatan, kita hitung perlahan...
    let hasil = fibonacciCerdas(n - 1, cache) + fibonacciCerdas(n - 2, cache);
    
    // 4. SIMPAN hasilnya ke buku catatan sebelum dikembalikan!
    cache[n] = hasil; 
    
    return hasil;
}

// Menjalankan fibonacciCerdas(50) akan selesai dalam 0.001 detik!
console.log(fibonacciCerdas(50)); // Output: 12586269025
```

## 5. Koneksi dengan Web Development

Pemikiran _Dynamic Programming_ dan _Memoization_ sangat sering dipakai oleh _Frontend_ dan _Backend Developer_ tingkat lanjut:

1. **React `useMemo` & `React.memo`:** Jika Anda menggunakan _framework_ React, Anda akan sering bertemu _Hooks_ bernama `useMemo`. Ini adalah penerapan harfiah dari Memoisasi! Jika ada komponen tabel data yang sangat berat untuk di-_render_, Anda bisa "mengingat" (_memoize_) tabel tersebut sehingga _browser_ tidak perlu merakit ulang HTML-nya setiap kali pengguna menekan tombol lain di halaman tersebut.
    
2. **API Caching di Backend (Redis):** Saat jutaan orang membuka halaman utama _e-commerce_, _Backend Developer_ tidak menyuruh _database_ mencari daftar produk dari nol setiap detik (itu akan membuat _server crash_). Mereka melakukan _query_ sekali, menyimpan hasilnya di sistem _cache_ (seperti Redis), dan memberikan "contekannya" kepada jutaan pengunjung berikutnya dengan kecepatan kilat.
    
3. **Optimasi Game Web:** Kalkulasi pergerakan karakter, tabrakan (_collision_), atau skor yang memiliki pola berulang selalu di-_cache_ agar _Frame Rate_ (FPS) game di _browser_ tetap stabil di angka 60 FPS tanpa membuat laptop panas.
