## 1. Definisi Singkat

**Merge Sort** dan **Quick Sort** adalah algoritma pengurutan tingkat lanjut yang menyelesaikan masalah dengan cara memecah _Array_ besar menjadi bagian-bagian yang sangat kecil, mengurutkan bagian kecil tersebut, lalu menggabungkannya kembali menjadi satu kesatuan yang utuh. Keduanya sangat mengandalkan konsep **Fungsi Rekursif** (fungsi yang memanggil dirinya sendiri) yang sudah kita pelajari sebelumnya.

## 2. Analogi: Menyortir Ujian dan Membariskan Siswa

Bayangkan Anda seorang guru yang harus mengurutkan 1.000 lembar kertas ujian berdasarkan abjad:

- **Merge Sort (Pecah dan Gabung):** Anda membagi tumpukan 1.000 kertas itu menjadi dua (500-500). Lalu membaginya lagi (250-250), dan terus membaginya sampai setiap tumpukan hanya berisi **1 kertas** (sudah pasti urut, kan?). Setelah itu, Anda mulai **menggabungkan (Merge)** tumpukan 1 kertas menjadi 2 kertas secara berurutan. Lalu menumpuk 2 kertas menjadi 4 kertas secara berurutan. Terus dilakukan sampai 1.000 kertas kembali bersatu dengan rapi.
    
- **Quick Sort (Pilih Bos / Pivot):** Anda menyuruh 1.000 siswa berbaris di lapangan. Anda memilih satu siswa secara acak sebagai **"Bos" (Pivot)**. Anda berteriak: _"Yang lebih pendek dari Bos, lari ke kiri! Yang lebih tinggi, lari ke kanan!"_. Sekarang lapangan terbagi dua. Anda lalu memilih "Bos" baru di kelompok kiri dan "Bos" baru di kelompok kanan, dan mengulangi perintah yang sama. Proses ini diulang sampai semua kelompok hanya tersisa 1 orang, dan tiba-tiba seluruh lapangan sudah berbaris rapi!
    

## 3. Poin-Poin Penting

- **Kompleksitas Waktu Sangat Cepat ($O(n \log n)$):** Inilah alasan mereka disebut tingkat dewa. Jika Anda mengurutkan 1 juta data, algoritma lama $O(n^2)$ butuh 1 triliun langkah. Merge/Quick Sort hanya butuh sekitar **20 juta langkah**. Perbedaannya bisa berarti waktu tunggu 2 jam vs 0,1 detik!
    
- **Merge Sort (Stabil tapi Boros Memori):** Selalu konsisten berjalan di kecepatan $O(n \log n)$ apa pun yang terjadi. Namun, ia butuh memori RAM tambahan ($O(n)$) untuk membuat _Array_ penampung saat proses penggabungan.
    
- **Quick Sort (Gesit dan Hemat Memori):** Ia melakukan pengurutan langsung di tempat tanpa memakan RAM tambahan. Namun, jika Anda kebetulan memilih "Bos" (Pivot) yang paling salah terus-menerus (skenario terburuk), kecepatannya bisa anjlok menjadi $O(n^2)$. Meski begitu, dalam praktiknya, Quick Sort seringkali lebih cepat dari Merge Sort.
    

## 4. Contoh Implementasi

Mari kita lihat betapa elegannya **Quick Sort** saat ditulis menggunakan bahasa **JavaScript** modern. Perhatikan bagaimana ia memanggil dirinya sendiri (Rekursi):

JavaScript

```
function quickSort(array) {
    // 1. BASE CASE (Berhenti pecah kalau isi array sisa 1 atau kosong)
    if (array.length <= 1) {
        return array;
    }

    // 2. Tentukan "Bos" (Pivot). Kita ambil elemen paling belakang.
    let pivot = array[array.length - 1];
    
    // Siapkan dua kubu (kiri dan kanan)
    let kubuKiri = [];
    let kubuKanan = [];

    // 3. Pisahkan pasukan! (Looping ke semua elemen KECUALI si Pivot)
    for (let i = 0; i < array.length - 1; i++) {
        if (array[i] < pivot) {
            kubuKiri.push(array[i]);  // Lebih kecil masuk kiri
        } else {
            kubuKanan.push(array[i]); // Lebih besar masuk kanan
        }
    }

    // 4. REKURSI: Ulangi proses ke kubu kiri & kanan, lalu gabungkan dengan Pivot di tengah!
    return [...quickSort(kubuKiri), pivot, ...quickSort(kubuKanan)];
}

let angkaAcak = [44, 12, 59, 9, 21, 88, 3];
console.log("Setelah Quick Sort:", quickSort(angkaAcak));
// Output: [3, 9, 12, 21, 44, 59, 88]
```

## 5. Koneksi dengan Web Development

Anda mungkin tidak akan pernah menulis manual kode _Quick Sort_ di proyek web nyata, namun algoritma ini bekerja tanpa henti di belakang layar:

1. **Mesin JavaScript (V8 Engine Chrome):** Saat Anda menggunakan metode bawaan `array.sort()` di JavaScript, _browser_ Anda sebenarnya mengeksekusi algoritma canggih bernama **Timsort** (gabungan _Merge Sort_ dan _Insertion Sort_), atau versi adaptasi dari _Quick Sort_.
    
2. **Database Indexing:** _Backend Developer_ yang menarik jutaan data dari PostgreSQL sering meminta data tersebut diurutkan berdasar tanggal terbaru (`ORDER BY date DESC`). Mesin _database_ menggunakan algoritma $O(n \log n)$ ini agar _server_ tidak meledak saat melayani permintaan tersebut.
    
3. **Wawancara Kerja Skala Global:** Jika Anda melamar posisi _Software Engineer_ di perusahaan multinasional, Anda hampir 100% akan diminta menjelaskan cara kerja _Merge Sort_ atau _Quick Sort_ di papan tulis dan membedah Big-O _Notation_-nya.
