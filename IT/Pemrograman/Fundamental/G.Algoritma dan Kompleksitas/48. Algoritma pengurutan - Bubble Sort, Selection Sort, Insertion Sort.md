Mari kita pecahkan masalah yang kita temui di materi sebelumnya. _Binary Search_ memang sangat cepat, tetapi ia menuntut satu syarat mutlak: **datanya harus sudah berurutan**.

Lalu, bagaimana cara komputer mengurutkan ribuan data acak dari nilai terkecil ke terbesar? Di sinilah Algoritma Pengurutan (_Sorting Algorithms_) mengambil panggung. Kita akan mulai dari tiga algoritma paling dasar dan klasik.

---

## 1. Definisi Singkat

**Sorting Algorithms** adalah serangkaian instruksi untuk menyusun elemen-elemen dalam sebuah kumpulan data (seperti _Array_) ke dalam urutan tertentu, biasanya secara _Ascending_ (A-Z / 0-9) atau _Descending_ (Z-A / 9-0).

Tiga algoritma dasar yang wajib dipelajari pemula adalah:

1. **Bubble Sort:** Menggelembungkan data terbesar ke ujung kanan dengan membandingkan dua data yang bersebelahan.
    
2. **Selection Sort:** Memilih data terkecil dari seluruh kumpulan, lalu menukarnya ke posisi paling depan.
    
3. **Insertion Sort:** Menyisipkan setiap data satu per satu ke posisi yang tepat di bagian kiri yang sudah dianggap berurutan.
    

## 2. Analogi: Merapikan Barisan dan Bermain Kartu

Bayangkan Anda ditugaskan mengurutkan tinggi badan 10 siswa dari yang terpendek hingga tertinggi:

- **Bubble Sort (Tukar Sebelah):** Anda berjalan dari kiri ke kanan. Anda membandingkan Siswa 1 dan Siswa 2. Jika Siswa 1 lebih tinggi, mereka **bertukar tempat**. Lalu Anda membandingkan Siswa 2 dan Siswa 3, tukar lagi jika perlu. Proses ini diulang terus sampai siswa tertinggi "menggelembung" ke ujung kanan barisan.
    
- **Selection Sort (Cari yang Terpendek):** Anda memindai seluruh barisan dari ujung ke ujung hanya untuk mencari **satu siswa terpendek**. Setelah ketemu, Anda menyuruhnya pindah ke urutan nomor 1. Lalu Anda memindai sisa 9 siswa untuk mencari yang terpendek kedua, dan menaruhnya di urutan nomor 2. Begitu seterusnya.
    
- **Insertion Sort (Seperti Main Kartu Remi):** Bayangkan Anda sedang memegang kartu remi di tangan kiri. Setiap kali Anda mengambil kartu baru dari meja dengan tangan kanan, Anda langsung **menyisipkannya** ke posisi yang tepat di antara kartu-kartu di tangan kiri Anda agar selalu berurutan.
    

## 3. Poin-Poin Penting

- **Kompleksitas Waktu O(n²):** Ketiga algoritma ini masuk dalam kasta **O(n²)** (Kuadratik) untuk skenario terburuknya. Artinya, mereka menggunakan perulangan bersarang (_nested loops_). Jika ada 100.000 data, komputer bisa melakukan 10 miliar kali pengecekan. Sangat lambat untuk data raksasa!
    
- **Hemat Memori O(1):** Kelebihan ketiganya adalah mereka melakukan pengurutan langsung di dalam wadah aslinya (_in-place sorting_). Mereka tidak butuh memori RAM tambahan untuk membuat wadah _Array_ tiruan.
    
- **Kapan Digunakan?** _Bubble_ dan _Selection_ jarang dipakai di industri modern karena terlalu lambat. Namun, **Insertion Sort** secara mengejutkan sangat cepat (mendekati O(n)) jika digunakan pada data yang jumlahnya sedikit atau data yang _hampir_ berurutan.
    

## 4. Contoh Implementasi

Mari kita lihat implementasi **Bubble Sort**, algoritma pengurutan yang paling ikonik di kalangan _programmer_ pemula menggunakan **JavaScript**:

JavaScript

```
let dataAcak = [5, 3, 8, 4, 2];

function bubbleSort(array) {
    let panjang = array.length;
    
    // Perulangan luar: Mengulang proses sebanyak jumlah elemen
    for (let i = 0; i < panjang; i++) {
        
        // Perulangan dalam: Membandingkan elemen bersebelahan
        // (panjang - i - 1) karena elemen terbesar sudah pasti ada di ujung kanan setiap putaran selesai
        for (let j = 0; j < panjang - i - 1; j++) {
            
            // Jika elemen sebelah kiri LEBIH BESAR dari sebelah kanan, TUKAR!
            if (array[j] > array[j + 1]) {
                let simpananSementara = array[j];
                array[j] = array[j + 1];
                array[j + 1] = simpananSementara;
            }
        }
    }
    return array;
}

console.log("Data Sebelum:", dataAcak); 
// Output: [5, 3, 8, 4, 2]

console.log("Data Sesudah:", bubbleSort(dataAcak)); 
// Output: [2, 3, 4, 5, 8]
```
