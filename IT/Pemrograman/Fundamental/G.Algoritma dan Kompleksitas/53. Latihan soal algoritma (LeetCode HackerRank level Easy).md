## 3. Soal 1: Two Sum (LeetCode #1)

Ini adalah soal nomor satu di LeetCode dan merupakan "Pintu Gerbang" dunia algoritma.

**Deskripsi Masalah:**

Diberikan sebuah _Array_ berisi angka (`nums`) dan sebuah angka target (`target`). Kembalikan **indeks** dari dua angka di dalam _Array_ tersebut yang jika dijumlahkan hasilnya sama dengan `target`. Anda tidak boleh menggunakan elemen yang sama dua kali.

**Contoh:**

- Input: `nums = [2, 7, 11, 15]`, `target = 9`
    
- Output: `[0, 1]` (Karena `nums[0] + nums[1] == 2 + 7 == 9`)
    

**Solusi & Pembahasan (JavaScript):**

Jika Anda menggunakan dua _looping_ bersarang (Mengecek 2+7, lalu 2+11, dst), itu adalah cara _Brute Force_ dengan kompleksitas waktu $O(n^2)$. Kita bisa membuatnya secepat kilat ($O(n)$) menggunakan objek/Hash Map sebagai "Buku Catatan" (mengingat pelajaran _Dynamic Programming_ kita!).

JavaScript

```
function twoSum(nums, target) {
    // Siapkan "Buku Catatan" kosong
    let catatan = {}; 

    for (let i = 0; i < nums.length; i++) {
        let angkaSaatIni = nums[i];
        
        // Berapa angka yang kita butuhkan agar mencapai target?
        let angkaYangDicari = target - angkaSaatIni; 

        // Cek apakah angka yang dibutuhkan ADA di dalam buku catatan kita
        if (angkaYangDicari in catatan) {
            // Jika ada, kembalikan indeks angka yang dicari, dan indeks angka saat ini
            return [catatan[angkaYangDicari], i];
        }

        // Jika tidak ada, catat angka saat ini beserta indeksnya, lalu lanjut ke angka berikutnya
        catatan[angkaSaatIni] = i; 
    }
    
    return []; // Jika tidak ditemukan
}

console.log(twoSum([2, 7, 11, 15], 9)); // Output: [0, 1]
```

## 4. Soal 2: Valid Palindrome (LeetCode #125)

Soal ini menguji seberapa lihai Anda memanipulasi teks (_String_) dan memperkenalkan teknik yang sangat populer: **Two Pointers (Dua Penunjuk)**.

**Deskripsi Masalah:**

Sebuah kalimat disebut _Palindrome_ jika setelah semua huruf diubah menjadi huruf kecil dan semua spasi/simbol dihapus, kalimat tersebut dibaca sama baik dari depan maupun dari belakang.

**Contoh:**

- Input: `s = "A man, a plan, a canal: Panama"`
    
- Output: `true` (Karena dibaca "amanaplanacanalpanama" dari kiri ke kanan atau kanan ke kiri tetap sama).
    

**Solusi & Pembahasan (JavaScript):**

JavaScript

```
function isPalindrome(s) {
    // 1. BERSINKAN DATA: Ubah ke huruf kecil dan buang karakter non-alfanumerik
    // Kita gunakan Regular Expression (Regex) yang sudah standar di industri
    let bersih = s.toLowerCase().replace(/[^a-z0-9]/g, '');

    // 2. TEKNIK TWO POINTERS: Satu orang tunjuk dari ujung kiri, satu dari ujung kanan
    let kiri = 0;
    let kanan = bersih.length - 1;

    // Selama penunjuk kiri belum melewati penunjuk kanan
    while (kiri < kanan) {
        // Jika huruf di kiri BEDA dengan huruf di kanan, pasti BUKAN palindrome!
        if (bersih[kiri] !== bersih[kanan]) {
            return false;
        }
        // Jika sama, geser kedua penunjuk ke arah tengah
        kiri++;
        kanan--;
    }

    // Jika berhasil sampai ke tengah tanpa masalah, berarti Palindrome!
    return true; 
}

console.log(isPalindrome("A man, a plan, a canal: Panama")); // Output: true
console.log(isPalindrome("race a car")); // Output: false
```

## 5. Tips Sukses Berlatih Algoritma

Jika Anda baru memulai perjalanan di LeetCode atau HackerRank, pegang 3 aturan emas ini:

1. **Aturan 20 Menit:** Jika Anda mentok memikirkan solusi selama lebih dari 20-30 menit, **berhentilah dan lihat solusinya**. Jangan buang waktu 3 jam untuk memandangi layar kosong. Anda sedang belajar pola, bukan menemukan teori matematika baru.
    
2. **Jangan Menghafal Kode:** Pahami _mengapa_ solusi tersebut menggunakan _Hash Map_ atau _Two Pointers_. Jika Anda hanya menghafal ketikannya, Anda akan blank saat soalnya sedikit diubah.
    
3. **Lacak Kompleksitasnya:** Biasakan diri Anda untuk selalu bisa menjawab: _"Apa Time Complexity dan Space Complexity dari kode yang baru saja saya tulis?"_
