## 1. Definisi Singkat

**Algoritma Pencarian (Search Algorithms)** adalah serangkaian instruksi langkah demi langkah yang digunakan komputer untuk menemukan letak suatu data spesifik di dalam sebuah struktur data (seperti Array). Dua metode yang paling fundamental adalah:

- **Linear Search (Pencarian Berurutan):** Komputer mengecek setiap elemen satu per satu dari ujung kiri sampai ke ujung kanan hingga data ditemukan.
    
- **Binary Search (Pencarian Bagi Dua):** Komputer menebak nilai tengah dari sebuah kumpulan data yang **sudah berurutan**, lalu membuang setengah bagian yang pasti salah, dan mengulangi proses tersebut hingga data ditemukan.
    

## 2. Analogi: Mencari Kata di Kamus Fisik

Bayangkan Anda sedang mencari arti kata "PEMROGRAMAN" di dalam sebuah Kamus Bahasa Indonesia tebal berisi 10.000 halaman:

- **Linear Search:** Anda membuka halaman 1, membaca kata pertama. Salah. Lanjut kata kedua. Salah. Lanjut terus satu per satu sampai halaman 5.000. Ini adalah cara yang sangat lambat dan melelahkan, layaknya algoritma berkecepatan $O(n)$.
    
- **Binary Search:** Anda langsung membelah kamus tepat di tengah (halaman 5.000), misalnya terbuka huruf "M". Karena Anda tahu huruf "P" letaknya pasti _setelah_ huruf "M", Anda langsung **membuang/mengabaikan** paruh pertama kamus (halaman 1 sampai 5.000). Kini Anda hanya fokus membelah tengah lagi di sisa halaman (5.001 - 10.000). Dalam hitungan kurang dari 15 kali belah, Anda pasti menemukan kata "PEMROGRAMAN". Ini adalah kecepatan $O(\log n)$ yang luar biasa!
    

## 3. Poin-Poin Penting

- **Syarat Mutlak Binary Search:** Binary Search **TIDAK AKAN BISA** bekerja jika data Anda masih acak-acakan (belum diurutkan dari kecil ke besar atau A ke Z). Jika data acak, Anda terpaksa menggunakan Linear Search.
    
- **Kompleksitas Waktu (Time Complexity):** * Linear Search memiliki skenario terburuk $O(n)$. Jika ada 1 juta data, komputer bisa saja melakukan 1 juta pengecekan.
    
    - Binary Search memiliki skenario terburuk $O(\log n)$. Jika ada 1 juta data, komputer **maksimal hanya butuh 20 kali pengecekan** untuk menemukan datanya!
        
- **Metode Bawaan Bahasa (Built-in Methods):** Di JavaScript, metode seperti `array.indexOf()`, `array.find()`, atau `array.includes()` di belakang layar sebenarnya menggunakan algoritma Linear Search.
    

## 4. Contoh Implementasi

Mari kita lihat perbandingan kedua algoritma ini dalam bentuk kode **JavaScript**:

JavaScript

```
let daftarAngka = [2, 5, 8, 12, 16, 23, 38, 56, 72, 91]; // Data SUDAH berurutan

// ==========================================
// 1. LINEAR SEARCH (O(n))
// Mengecek satu per satu dari awal
// ==========================================
function linearSearch(array, target) {
    for (let i = 0; i < array.length; i++) {
        if (array[i] === target) {
            return `Ketemu di indeks ${i} (butuh ${i + 1} langkah)`;
        }
    }
    return "Data tidak ditemukan";
}

console.log("Linear:", linearSearch(daftarAngka, 72)); 
// Output: Linear: Ketemu di indeks 8 (butuh 9 langkah)


// ==========================================
// 2. BINARY SEARCH (O(log n))
// Membelah array menjadi dua terus-menerus
// ==========================================
function binarySearch(array, target) {
    let kiri = 0;
    let kanan = array.length - 1;
    let langkah = 0;

    while (kiri <= kanan) {
        langkah++;
        // Mencari titik tengah
        let tengah = Math.floor((kiri + kanan) / 2);

        if (array[tengah] === target) {
            return `Ketemu di indeks ${tengah} (butuh ${langkah} langkah)`;
        } else if (array[tengah] < target) {
            kiri = tengah + 1; // Buang bagian kiri
        } else {
            kanan = tengah - 1; // Buang bagian kanan
        }
    }
    return "Data tidak ditemukan";
}

console.log("Binary:", binarySearch(daftarAngka, 72)); 
// Output: Binary: Ketemu di indeks 8 (butuh 2 langkah saja!)
```

## 5. Koneksi dengan Web Development

Pemahaman algoritma pencarian ini sangat aplikatif dalam membangun _website_ yang responsif:

1. **Mencari Produk (Frontend):** Jika Anda membuat aplikasi Kasir Kasar (Point of Sale) yang menyimpan 50 data menu di _browser_ (jumlahnya sedikit), Anda cukup menggunakan `array.find()` (Linear Search) untuk memproses pencarian saat kasir mengetik nama menu. Komputer modern bisa memproses 50 Linear Search tanpa _lag_.
    
2. **Database & Mesin Pencari (Backend):** Saat pengguna mencari tiket pesawat dari 100 juta jadwal penerbangan, _database_ di _server_ (seperti PostgreSQL atau Elasticsearch) menggunakan modifikasi tingkat tinggi dari Binary Search (seperti _B-Trees_) agar hasil pencarian tiket bisa muncul dalam hitungan milidetik.
    
3. **Git Bisect (Debugging):** Saat kode _website_ Anda tiba-tiba _error_ dan Anda memiliki 500 riwayat penyimpanan kode (Commit) di Git, alat bernama `git bisect` akan menggunakan algoritma Binary Search untuk melompat ke tengah-tengah riwayat kode dan mencari tahu persis di kode versi mana _error_ tersebut mulai terjadi, menghemat jam kerja Anda.
