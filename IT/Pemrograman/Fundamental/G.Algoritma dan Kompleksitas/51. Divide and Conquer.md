## 1. Definisi Singkat

**Divide and Conquer (Pecah dan Taklukkan)** adalah paradigma algoritma yang memecah sebuah masalah besar dan kompleks menjadi beberapa sub-masalah yang lebih kecil dan sejenis. Setelah sub-masalah ini berukuran cukup kecil hingga sangat mudah diselesaikan secara langsung, hasil dari sub-masalah tersebut akan digabungkan kembali untuk membentuk solusi akhir dari masalah aslinya.

## 2. Analogi: Menghitung 10.000 Koin Emas

Bayangkan Anda adalah seorang bendahara kerajaan yang diberi satu peti raksasa berisi 10.000 koin emas dan diminta untuk menghitung totalnya dalam waktu singkat.

- **Cara Biasa (Linear):** Anda menghitungnya satu per satu dari koin ke-1 sampai ke-10.000. Anda akan kelelahan, memakan waktu seharian, dan rentan salah hitung.
    
- **Cara Divide and Conquer:**
    
    1. **Divide (Pecah):** Anda membagi tumpukan koin itu ke 10 menteri (masing-masing mendapat tumpukan 1.000 koin). Para menteri membaginya lagi ke 10 asisten (masing-masing 100 koin).
        
    2. **Conquer (Taklukkan):** Para asisten menghitung 100 koin tersebut. (Menghitung 100 koin sangatlah mudah, cepat, dan minim kesalahan). Ini disebut _Base Case_.
        
    3. **Combine (Gabungkan):** Asisten melaporkan jumlahnya ke menteri (100 x 10 = 1.000). Menteri melaporkannya kepada Anda (1.000 x 10 = 10.000).
        

Masalah raksasa selesai dalam hitungan menit berkat strategi delegasi!

## 3. Poin-Poin Penting

Algoritma ini selalu berputar pada tiga langkah suci:

- **Divide:** Membagi masalah menjadi sub-masalah yang lebih kecil. (Biasanya dengan membelah himpunan data menjadi dua bagian yang sama besar).
    
- **Conquer:** Menyelesaikan sub-masalah secara rekursif. Jika sub-masalah sudah sangat kecil (hanya 1 atau 2 elemen), langsung selesaikan saat itu juga.
    
- **Combine:** Menggabungkan hasil dari sub-masalah tersebut menjadi solusi akhir.
    
- **Sahabat Karib Rekursi:** Hampir mustahil mengimplementasikan _Divide and Conquer_ tanpa menggunakan Fungsi Rekursif (fungsi yang memanggil dirinya sendiri), karena kita harus terus membelah data sampai mencapai unit terkecil.
    

## 4. Contoh Implementasi

Mari kita buat contoh sederhana: **Menjumlahkan seluruh angka di dalam sebuah Array**. Alih-alih menggunakan _looping_ biasa, kita akan mendemonstrasikan gaya pemecahan masalah _Divide and Conquer_ menggunakan **JavaScript**:

JavaScript

```
function jumlahkanArrayDC(array) {
    // 2. CONQUER (Base Case / Kondisi Berhenti)
    // Jika array kosong, jumlahnya 0
    if (array.length === 0) return 0;
    // Jika array sisa 1 elemen, kembalikan elemen tersebut (sangat mudah!)
    if (array.length === 1) return array[0];

    // 1. DIVIDE (Pecah array menjadi dua bagian)
    let tengah = Math.floor(array.length / 2);
    let kubuKiri = array.slice(0, tengah);  // Ambil potongan kiri
    let kubuKanan = array.slice(tengah);    // Ambil potongan kanan

    // 3. COMBINE (Gabungkan hasil perhitungan rekursif dari kiri dan kanan)
    return jumlahkanArrayDC(kubuKiri) + jumlahkanArrayDC(kubuKanan);
}

let daftarAngka = [10, 20, 30, 40, 50, 60];

console.log("Total Jumlah:", jumlahkanArrayDC(daftarAngka));
// Output: Total Jumlah: 210

/*
Proses di balik layar:
1. [10, 20, 30] ditambah [40, 50, 60]
2. ([10] + [20, 30]) ditambah ([40] + [50, 60])
3. ([10] + ([20] + [30])) ditambah ([40] + ([50] + [60]))
4. Semua bagian terkecil sudah berwujud angka tunggal, lalu ditambahkan ke atas!
*/
```

## 5. Koneksi dengan Web Development

Pemikiran _Divide and Conquer_ adalah kunci untuk memproses data masif di dunia industri teknologi:

1. **Mengunggah Video Besar (Chunking Uploads):** Saat Anda mengunggah video 2GB ke YouTube, _browser_ tidak mengirim file itu utuh (rentan gagal di tengah jalan). _File_ dipecah menjadi ribuan blok kecil berukuran 5MB (_Divide_), diunggah secara terpisah (_Conquer_), dan disatukan kembali oleh _server_ YouTube (_Combine_).
    
2. **Sistem Terdistribusi (Big Data):** Di _Backend_, perusahaan seperti Google atau Tokopedia menggunakan alat seperti Hadoop (_MapReduce_). Jika mereka harus menganalisis 1 Terabyte data transaksi, data itu dipecah dan dikirim ke 100 _server_ berbeda untuk dihitung secara bersamaan, lalu hasilnya direkap di satu _server_ induk. Sangat cepat!
    
3. **Parallel Processing (Web Workers):** Di _Frontend_, jika ada kalkulasi matematika berat yang bisa membekukan tampilan UI _browser_, _developer_ akan memecah kalkulasi tersebut dan melemparnya ke _Web Workers_ (utas pekerja di latar belakang browser), lalu menggabungkan hasilnya saat sudah selesai tanpa mengganggu layar pengguna.
