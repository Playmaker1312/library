Mari kita berkenalan dengan salah satu pola pikir algoritma yang paling praktis, cepat, tetapi juga sedikit "egois". Jika algoritma sebelumnya memecah masalah dengan sangat hati-hati, algoritma ini mengambil keputusan instan berdasarkan apa yang terlihat paling menguntungkan saat itu juga.

---

## 1. Definisi Singkat

**Algoritma Greedy (Serakah)** adalah sebuah pendekatan pemecahan masalah yang selalu membuat pilihan terbaik pada saat itu juga (**Local Optimum**), dengan harapan bahwa serangkaian pilihan "egois" tersebut akan menuntun pada solusi terbaik secara keseluruhan (**Global Optimum**).

Prinsip utamanya adalah: _Ambil apa yang paling menguntungkan sekarang, dan jangan pernah menyesali atau melihat kembali keputusan yang sudah dibuat._

## 2. Analogi: Kasir Minimarket Memberi Kembalian

Bayangkan Anda adalah kasir yang harus memberi uang kembalian sebesar **Rp 6.500** kepada pelanggan, dan Anda ingin memberikan **jumlah koin/lembar sesedikit mungkin** agar dompet pelanggan tidak berat.

Pecahan yang tersedia: Rp 5.000, Rp 2.000, Rp 1.000, dan Rp 500.

- **Langkah Greedy 1:** Anda melihat pecahan terbesar. Apakah Rp 5.000 bisa dipakai? Ya! Sisa kembalian: Rp 1.500.
    
- **Langkah Greedy 2:** Pecahan terbesar berikutnya yang muat adalah Rp 1.000. Ambil! Sisa kembalian: Rp 500.
    
- **Langkah Greedy 3:** Pecahan Rp 500 muat. Ambil! Sisa kembalian: Rp 0.
    

Total uang yang diberikan: 3 lembar/koin (5000, 1000, 500). Ini adalah cara kerja algoritma Greedy! Sangat cepat dan langsung pada tujuannya tanpa berpikir panjang.

## 3. Poin-Poin Penting

- **Sangat Cepat dan Efisien:** Karena tidak pernah mengevaluasi ulang keputusan yang sudah diambil (tidak ada _backtracking_), algoritma ini berjalan dengan kecepatan kilat, seringkali _O(n)_ atau _O(n log n)_.
    
- **Kelemahan Fatal (Tidak Selalu Optimal):** Algoritma ini bisa "terjebak". Bayangkan Anda harus memberi kembalian **Rp 6.000**, tapi pecahan yang tersedia hanya **Rp 4.000, Rp 3.000, dan Rp 1.000**.
    
    - _Cara Greedy:_ Ambil pecahan terbesar dulu (4.000). Sisa 2.000. Lalu ambil dua koin 1.000. Total = **3 koin** (4000, 1000, 1000).
        
    - _Cara Cerdas (Optimal):_ Ambil dua koin 3.000. Total = **2 koin** (3000, 3000). Di skenario ini, algoritma Greedy **gagal** memberikan solusi terbaik!
        
- **Kapan Digunakan:** Digunakan ketika masalah tersebut memiliki sifat di mana pilihan lokal yang optimal memang terbukti secara matematis akan menghasilkan solusi global yang optimal.
    

## 4. Contoh Implementasi

Mari kita tulis algoritma Kasir Minimarket tadi menggunakan **JavaScript**:

JavaScript

```
function kembalianGreedy(jumlahKembalian) {
    // Daftar koin harus diurutkan dari yang terbesar!
    let koinTersedia = [5000, 2000, 1000, 500, 200, 100];
    let hasilKoin = [];
    let sisa = jumlahKembalian;

    for (let i = 0; i < koinTersedia.length; i++) {
        let koinSaatIni = koinTersedia[i];
        
        // Selama sisa kembalian masih cukup untuk dikurangi koin ini
        while (sisa >= koinSaatIni) {
            hasilKoin.push(koinSaatIni); // Masukkan koin ke kantong
            sisa = sisa - koinSaatIni;   // Kurangi sisa kembalian
        }
        
        // Jika sisa sudah 0, hentikan proses (efisiensi!)
        if (sisa === 0) break;
    }

    return hasilKoin;
}

let kembalianBudi = 6700;
console.log(`Kembalian untuk Rp ${kembalianBudi}:`, kembalianGreedy(kembalianBudi));
// Output: [5000, 1000, 500, 200] (Total 4 koin)
```

## 5. Koneksi dengan Web Development

Meskipun terdengar seperti teori matematika, algoritma Greedy bekerja setiap saat di balik layar dunia internet:

1. **Kompresi Data (Huffman Coding):** Saat Anda mengunggah gambar atau video ke _website_, ukurannya dikompresi agar _loading_ web lebih cepat. Algoritma kompresi (seperti format PNG atau algoritma gzip) menggunakan pendekatan Greedy untuk memberikan kode biner terpendek pada piksel atau huruf yang paling sering muncul.
    
2. **Aplikasi Navigasi Peta (Dijkstra):** Saat Anda menggunakan Google Maps atau Gojek, algoritma (seperti algoritma Dijkstra) menggunakan prinsip Greedy untuk selalu mencari simpang jalan terdekat berikutnya yang bisa dijangkau, demi merangkai rute terpendek dari rumah Anda ke tempat tujuan.
    
3. **Penjadwalan Server (Task Scheduling):** _Server cloud_ besar (seperti AWS atau Google Cloud) menggunakan Greedy untuk mengatur ribuan permintaan web yang masuk. Ia akan memprioritaskan "tugas yang paling cepat selesai lebih dulu" agar antrean tidak macet.
