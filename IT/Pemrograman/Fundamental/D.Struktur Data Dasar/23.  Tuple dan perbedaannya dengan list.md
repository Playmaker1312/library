## 1. Definisi Singkat

**Tuple** adalah struktur data yang mirip dengan Array/List, berfungsi untuk menyimpan sekumpulan nilai dalam satu variabel. Perbedaan utamanya: Tuple bersifat **Immutable (Kekal / Tidak dapat diubah)**. Artinya, setelah sebuah Tuple dibuat, Anda sama sekali tidak bisa menambah, menghapus, atau mengubah elemen di dalamnya.

## 2. Analogi: Papan Tulis vs Prasasti Batu

Bayangkan Anda sedang membuat daftar instruksi:

- **Array / List** itu seperti **Papan Tulis**. Anda bisa menulis daftar tugas, lalu menghapus tugas nomor 2, atau menambahkan tugas baru di baris bawah kapan saja. Sifatnya fleksibel (_Mutable_).
    
- **Tuple** itu seperti **Prasasti Batu** atau **Dokumen Cetak**. Sekali hurufnya dipahat, Anda tidak bisa lagi menghapus atau menyisipkan huruf baru di tengah-tengahnya. Jika Anda benar-benar butuh teks yang berbeda, Anda harus menghancurkan batu tersebut dan memahat batu yang benar-benar baru (_Immutable_).
    

## 3. Poin-Poin Penting

- **Tidak Ada Operasi Update/Delete:** Anda bisa melakukan `Create` dan `Read`, tetapi Anda tidak bisa melakukan `Update` atau `Delete` pada elemen individu di dalam Tuple.
    
- **Lebih Cepat dan Ringan:** Karena komputer tahu ukuran Tuple tidak akan pernah berubah, komputer mengalokasikan memori yang lebih kecil dan bisa membacanya lebih cepat dibandingkan List.
    
- **Keamanan Data (Data Integrity):** Tuple digunakan untuk melindungi data agar tidak sengaja tertimpa atau diubah oleh fungsi lain di dalam program yang kompleks.
    
- **Perbedaan Sintaksis:** Di bahasa yang mendukung Tuple secara bawaan (seperti Python), List ditulis dengan kurung siku `[ ... ]`, sedangkan Tuple ditulis dengan kurung biasa `( ... )`.
    

## 4. Contoh Implementasi

Karena JavaScript murni tidak memiliki Tuple, mari kita lihat perbedaannya menggunakan bahasa **Python**, di mana konsep ini paling jelas terlihat:

Python

```
# 1. Ini adalah LIST (Menggunakan kurung siku)
koordinat_list = [10.5, 20.8]
koordinat_list[0] = 99.9  # SAH! Isi list sekarang berubah menjadi [99.9, 20.8]
koordinat_list.append(50) # SAH! Isi list bertambah menjadi [99.9, 20.8, 50]

# 2. Ini adalah TUPLE (Menggunakan kurung biasa)
koordinat_tuple = (10.5, 20.8)

# Jika kode di bawah ini dijalankan, program akan ERROR (Crash)
# koordinat_tuple[0] = 99.9  -> Error: Tuple object does not support item assignment
# koordinat_tuple.append(50) -> Error: Tuple object has no attribute 'append'
```

_Catatan untuk JS:_ Di JavaScript, _programmer_ sering "memalsukan" Tuple menggunakan Array biasa, atau menggunakan fungsi `Object.freeze([10.5, 20.8])` untuk membuat Array tersebut tidak bisa diubah lagi. Bahasa TypeScript (versi canggih dari JS) sudah memiliki fitur Tuple sungguhan.

## 5. Koneksi dengan Web Development

Dalam **Web Development**, konsep kekekalan (_immutability_) dari Tuple ini sangat sering diterapkan:

1. **React Hooks (`useState`):** Jika Anda nanti belajar _framework_ React.js, Anda akan sangat sering melihat sintaks `const [nilai, setNilai] = useState(0);`. Meskipun secara teknis itu adalah Array, secara konseptual itu adalah sebuah Tuple. Anda dijamin selalu mendapat 2 nilai: posisi pertama adalah datanya, posisi kedua adalah fungsi untuk mengubahnya. Jumlahnya tidak akan pernah bertambah jadi 3 atau berkurang jadi 1.
    
2. **Koordinat Peta (Maps API):** Saat aplikasi web Anda menggunakan Google Maps atau Mapbox, titik lokasi sering dikirim dan diterima dalam bentuk struktur menyerupai Tuple `(Latitude, Longitude)`. Karena titik koordinat adalah satu kesatuan mutlak, tidak masuk akal jika program tiba-tiba menambahkan elemen ke-3 ke dalamnya.
    
3. **Mengembalikan Banyak Nilai (Return Multiple Values):** Sebuah fungsi di _Backend_ (misal menggunakan Python Django) seringkali harus mengembalikan status sukses dan pesan error sekaligus. Alih-alih membuat _Class_ baru, _developer_ cukup mengembalikan Tuple: `return (True, "Data berhasil disimpan")`.
