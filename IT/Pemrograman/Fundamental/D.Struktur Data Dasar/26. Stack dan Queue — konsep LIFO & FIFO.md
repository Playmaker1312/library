## 1. Definisi Singkat

**Stack (Tumpukan)** dan **Queue (Antrean)** adalah struktur data linear yang mengatur urutan pemrosesan elemen.

- **Stack** menggunakan prinsip **LIFO (Last In, First Out)**: Data yang terakhir kali dimasukkan akan menjadi data pertama yang dikeluarkan.
    
- **Queue** menggunakan prinsip **FIFO (First In, First Out)**: Data yang pertama kali masuk akan menjadi data pertama yang diproses dan dikeluarkan.
    

## 2. Analogi: Tumpukan Piring vs Antrean Kasir

Bayangkan dua situasi ini:

- **Stack (LIFO):** Seperti **Tumpukan Piring** di restoran prasmanan. Karyawan menaruh piring bersih satu per satu ke atas tumpukan. Saat Anda mau makan, Anda pasti mengambil piring yang letaknya paling atas, bukan? Piring yang ditaruh paling akhir (Last In) adalah yang pertama diambil (First Out).
    
- **Queue (FIFO):** Seperti **Antrean Kasir** di minimarket. Orang pertama yang datang dan berdiri di barisan depan adalah orang pertama yang akan dilayani kasir. Orang yang datang terakhir harus berbaris di paling belakang dan menunggu gilirannya. Siapa cepat, dia dapat (First In, First Out).
    

## 3. Poin-Poin Penting

Dalam istilah pemrograman, aksi menambah dan menghapus data pada kedua struktur ini memiliki nama khusus:

- **Operasi pada Stack:**
    
    - `Push`: Menambahkan elemen baru ke posisi paling _atas_ (akhir).
        
    - `Pop`: Mengambil dan menghapus elemen dari posisi paling _atas_ (akhir).
        
- **Operasi pada Queue:**
    
    - `Enqueue`: Menambahkan elemen baru ke posisi paling _belakang_ antrean.
        
    - `Dequeue`: Mengambil dan menghapus elemen dari posisi paling _depan_ antrean.
        
- **Penggunaan:** Stack sangat cocok untuk membalikkan urutan atau merekam riwayat langkah mundur (_backtracking_). Queue sangat cocok untuk penjadwalan tugas (_scheduling_) yang membutuhkan keadilan urutan.
    

## 4. Contoh Implementasi

Dalam **JavaScript**, kita bisa dengan mudah menyimulasikan Stack dan Queue menggunakan Array bawaan dan metode khusus (_push_, _pop_, dan _shift_):

JavaScript

```
// ==========================================
// 1. SIMULASI STACK (LIFO - Last In, First Out)
// ==========================================
let riwayatBrowser = [];

// [PUSH] Menambah halaman baru (Masuk bertumpuk)
riwayatBrowser.push("Halaman Utama"); 
riwayatBrowser.push("Halaman Produk");
riwayatBrowser.push("Halaman Keranjang"); // Terakhir masuk

// [POP] Menekan tombol "Back" di browser
let halamanKembali = riwayatBrowser.pop(); 

console.log("Kembali dari: " + halamanKembali); // Output: "Halaman Keranjang"
console.log("Riwayat sisa: ", riwayatBrowser);  // Output: ["Halaman Utama", "Halaman Produk"]


// ==========================================
// 2. SIMULASI QUEUE (FIFO - First In, First Out)
// ==========================================
let antreanPesan = [];

// [ENQUEUE] Pesan masuk ke antrean
antreanPesan.push("Pesan dari Budi"); // Masuk pertama
antreanPesan.push("Pesan dari Andi");
antreanPesan.push("Pesan dari Siti"); // Masuk terakhir

// [DEQUEUE] Memproses pesan (Menggunakan shift() untuk mengambil dari depan)
let pesanDiproses = antreanPesan.shift(); 

console.log("Memproses: " + pesanDiproses); // Output: "Pesan dari Budi"
console.log("Antrean sisa: ", antreanPesan);  // Output: ["Pesan dari Andi", "Pesan dari Siti"]
```

## 5. Koneksi dengan Web Development

Dalam **Web Development**, konsep Stack dan Queue mengendalikan banyak hal di balik layar:

1. **Fitur Undo/Redo (Stack):** Saat Anda mengetik di editor web (seperti Google Docs) atau mendesain di Canva, setiap perubahan yang Anda lakukan di-_push_ ke dalam Stack. Saat Anda menekan `Ctrl + Z` (Undo), aplikasi melakukan _pop_ untuk mengembalikan keadaan web ke satu langkah sebelumnya.
    
2. **Sistem Antrean Email (Queue):** Saat _startup_ Anda mengirimkan email promosi ke 10.000 pengguna, _Backend Server_ tidak mengirimkannya sekaligus karena akan membuat _server crash_. Email tersebut di-_enqueue_ ke dalam _Message Queue_ (menggunakan alat seperti RabbitMQ atau Redis), lalu _server_ memprosesnya satu per satu (_dequeue_) secara berurutan.
    
3. **JavaScript Event Loop (Queue):** Ini adalah jantung dari JavaScript. Saat banyak animasi, klik tombol, dan pengambilan data internet terjadi bersamaan, JavaScript menaruh tugas-tugas itu di dalam antrean ( _Callback Queue_ / _Task Queue_) untuk dieksekusi satu per satu sesuai urutan masuknya, agar halaman web tidak _freeze_ (membeku).
