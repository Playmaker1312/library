## 1. Definisi Singkat

**Nested Conditions (Percabangan Bersarang)** adalah struktur di mana sebuah blok `if/else` diletakkan di dalam blok `if/else` lainnya untuk membuat pengecekan bersyarat yang bertingkat. **Nested Loops (Perulangan Bersarang)** adalah struktur di mana sebuah perulangan (`for` atau `while`) diletakkan di dalam perulangan lainnya, berguna untuk memproses data multi-dimensi atau tugas yang berulang dalam siklus.

## 2. Analogi: Jam Digital dan Pemeriksaan Bandara

- **Analogi Nested Loops (Jam Digital):** Bayangkan cara kerja jam dinding. Jarum jam (Loop Luar) menunjuk angka 1. Agar jarum jam bisa pindah ke angka 2, jarum menit (Loop Dalam) harus berputar penuh sebanyak 60 kali terlebih dahulu. Untuk setiap 1 langkah Loop Luar, Loop Dalam harus menyelesaikan seluruh putarannya.
    
- **Analogi Nested Conditions (Pemeriksaan Bandara):** Saat mau naik pesawat, Anda melewati pengecekan bertingkat.
    
    - _Pengecekan 1:_ "Apakah punya tiket?" (Jika Ya, lanjut ke dalam. Jika Tidak, pulang).
        
    - _Pengecekan 2 (Di dalam Pengecekan 1):_ "Apakah bawa barang berbahaya?" (Jika Ya, ditahan. Jika Tidak, boleh masuk pesawat). Anda tidak akan pernah ditanya soal barang berbahaya jika di pintu depan saja Anda tidak bawa tiket.
        

## 3. Poin-Poin Penting

- **Eksekusi Penuh Inner Loop:** Pada _nested loops_, perulangan yang ada di dalam (_inner loop_) akan diselesaikan dari awal hingga akhir untuk **setiap satu putaran** perulangan yang di luar (_outer loop_).
    
- **Hati-hati Kinerja Menurun:** Jika _outer loop_ berputar 10 kali, dan _inner loop_ berputar 10 kali, maka total instruksi yang dijalankan komputer adalah 10 x 10 = 100 kali. Semakin dalam sarangnya, aplikasi Anda akan semakin berat dan lambat (dalam ilmu komputer ini terkait dengan _Time Complexity_).
    
- **Keterbacaan Kode (Readability):** Jika Anda membuat _nested conditions_ atau _loops_ terlalu dalam (misalnya sampai 5 tingkat menjorok ke kanan), kode Anda akan terlihat seperti panah `>>>` dan sangat sulit dibaca oleh manusia. Ini sering disebut sebagai _Spaghetti Code_.
    
- **Variabel Independen:** Setiap loop bersarang harus menggunakan variabel penghitung yang berbeda (misalnya, loop luar menggunakan `i`, loop dalam menggunakan `j`) agar tidak saling bertabrakan.
    

## 4. Contoh Implementasi

Berikut adalah contoh penggunaan keduanya dalam **JavaScript**:

**1. Nested Conditions (Pengecekan Login Bertingkat):**

JavaScript

```
let username = "admin";
let password = "123";

// Loop Luar: Cek Username dulu
if (username === "admin") {
    
    // Loop Dalam: Hanya dieksekusi JIKA username benar
    if (password === "123") {
        console.log("Login sukses! Selamat datang Admin.");
    } else {
        console.log("Username benar, tapi password salah!");
    }

} else {
    console.log("Username tidak terdaftar.");
}
```

**2. Nested Loops (Mencetak Baris dan Kolom):**

JavaScript

```
// Loop Luar untuk mencetak 3 Baris
for (let baris = 1; baris <= 3; baris++) {
    let teksBaris = "Baris " + baris + " | Kolom: ";
    
    // Loop Dalam untuk mencetak 2 Kolom di SETIAP Baris
    for (let kolom = 1; kolom <= 2; kolom++) {
        teksBaris = teksBaris + kolom + " ";
    }
    
    console.log(teksBaris);
}
// Hasilnya:
// Baris 1 | Kolom: 1 2 
// Baris 2 | Kolom: 1 2 
// Baris 3 | Kolom: 1 2 
```

## 5. Koneksi dengan Web Development

Dalam **Web Development**, konsep bersarang ini adalah kunci untuk merender antarmuka yang kompleks:

1. **Membuat Tabel (Data Grid):** Saat _Frontend Developer_ ingin menampilkan data laporan keuangan dari _database_ ke dalam bentuk tabel HTML (`<table>`), mereka menggunakan _nested loops_. Loop luar untuk membuat baris ke bawah (`<tr>`), dan loop dalam untuk mengisi sel data ke samping (`<td>`).
    
2. **Validasi Formulir yang Kompleks:** Saat pengguna memilih metode pembayaran "Kartu Kredit" saat _checkout_, _nested conditions_ bekerja. _If_ (Metode == Kartu Kredit) -> _If_ (Nomor Kartu Valid) -> _If_ (Bulan Kadaluarsa Belum Lewat) -> Izinkan bayar.
    
3. **Sistem Dropdown Menu Bertingkat:** Navigasi web yang memiliki sub-menu (Kategori -> Pakaian -> Kemeja Pria) dibangun menggunakan data bersarang dan dirender menggunakan perulangan bersarang.
