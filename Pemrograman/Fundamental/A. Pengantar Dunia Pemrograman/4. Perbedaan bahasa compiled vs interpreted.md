## 1. Definisi Singkat

**Bahasa Compiled (Dikompilasi)** menerjemahkan seluruh _source code_ menjadi kode mesin secara utuh sebelum program dijalankan, menghasilkan file eksekusi (seperti `.exe`). Sebaliknya, **Bahasa Interpreted (Diinterpretasi)** menerjemahkan dan mengeksekusi _source code_ baris demi baris secara langsung pada saat program sedang berjalan (_runtime_).

## 2. Analogi: Menerjemahkan Buku Resep Asing

Bayangkan Anda mendapat sebuah buku resep dalam bahasa asing yang tidak Anda mengerti:

- **Compiled:** Anda menyewa penerjemah untuk menerjemahkan **seluruh isi buku** ke bahasa Indonesia, lalu mencetaknya menjadi buku baru. Butuh waktu lama di awal untuk menerjemahkan, tetapi setelah selesai, Anda bisa membacanya berulang kali dengan sangat cepat tanpa butuh si penerjemah lagi.
    
- **Interpreted:** Anda membawa penerjemah itu ke dapur. Ia membacakan dan menerjemahkan resep **kalimat demi kalimat** saat Anda sedang memasak. Anda bisa langsung mulai memasak detik itu juga, tapi proses memasak secara keseluruhan menjadi lebih lambat karena Anda harus terus menunggu terjemahan di setiap langkah.
    

## 3. Poin-Poin Penting

- **Kecepatan Eksekusi:** Program _compiled_ (seperti C, C++, Go) berjalan jauh lebih cepat karena sudah berupa bahasa mesin asli. Program _interpreted_ (seperti Python, PHP, Ruby) memiliki sedikit _overhead_ sehingga sedikit lebih lambat.
    
- **Proses Persiapan:** Bahasa _compiled_ membutuhkan tahap kompilasi yang memakan waktu (bisa bermenit-menit untuk aplikasi besar) sebelum bisa diuji coba. Bahasa _interpreted_ bisa langsung dijalankan seketika.
    
- **Portabilitas:** Kode _interpreted_ sangat fleksibel; satu kode bisa berjalan di Windows, Mac, atau Linux asalkan _interpreter_-nya terinstal. Kode _compiled_ harus dikompilasi ulang secara spesifik untuk masing-masing sistem operasi.
    
- **Pendekatan Hybrid (JIT):** Banyak teknologi modern menggunakan _Just-In-Time (JIT) Compilation_. Alih-alih murni diinterpretasi, kode diterjemahkan ke bahasa mesin tepat saat sedang dibutuhkan agar mendapat kecepatan layaknya bahasa _compiled_.
    

## 4. Contoh Implementasi

Perbedaan ini sangat terlihat pada cara kita menjalankan kodenya di terminal komputer:

**Bahasa Compiled (Contoh: C++)** Anda harus melakukan dua langkah. Pertama menerjemahkan (kompilasi), baru menjalankannya:

Bash

```
# 1. Proses kompilasi kode menjadi file eksekusi bernama 'aplikasi'
g++ main.cpp -o aplikasi

# 2. Menjalankan file eksekusi tersebut
./aplikasi
```

**Bahasa Interpreted (Contoh: Python)** Anda hanya butuh satu langkah, program langsung membaca dan menjalankan kodenya:

Bash

```
# Langsung jalankan kodenya menggunakan interpreter Python
python main.py
```

## 5. Koneksi dengan Web Development

Dalam bidang **Web Development**, kita menggunakan kedua jenis pendekatan ini secara berdampingan:

1. **Dunia Interpreted (JavaScript & PHP):** Sebagian besar web dibangun dengan bahasa _interpreted_. JavaScript yang berjalan di browser pengguna dibaca dan dieksekusi secara instan (dengan bantuan mesin JIT). Di sisi _server_, bahasa seperti PHP atau Node.js memproses permintaan pengguna baris demi baris secara dinamis.
    
2. **Membawa Kecepatan Compiled ke Web (WebAssembly):** Terkadang, web butuh performa tingkat tinggi yang tidak bisa diberikan oleh JavaScript (misalnya untuk game 3D di browser, editor video online, atau desain CAD). Web developer kini menggunakan teknologi **WebAssembly (Wasm)**, yang memungkinkan kita menulis program menggunakan bahasa _compiled_ (seperti C++ atau Rust), mengkompilasinya, dan menjalankannya langsung di dalam browser web dengan kecepatan _native_ (asli).
