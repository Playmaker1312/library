## 1. Definisi Singkat

**Pemrosesan Kode** adalah serangkaian tahap transformasi di mana instruksi yang ditulis manusia (_Source Code_) diterjemahkan menjadi bahasa biner 0 dan 1 (_Machine Code_) agar dapat dieksekusi secara fisik oleh CPU.

## 2. Analogi: Menerjemahkan Naskah Pidato

Bayangkan Anda adalah seorang **Presiden (Programmer)** yang menulis pidato dalam Bahasa Indonesia (**Source Code**). Namun, audiens Anda adalah **Robot (CPU)** yang hanya mengerti bahasa isyarat lampu mati/nyala (**Biner**).

Anda membutuhkan seorang **Penerjemah (Compiler/Interpreter)**. Penerjemah ini akan membaca naskah Anda, mengubah setiap kata menjadi pola gerakan lampu yang tepat, sehingga sang Robot bisa melakukan gerakan yang Anda perintahkan tanpa salah paham.

## 3. Poin-Poin Penting

Ada dua metode utama bagaimana komputer memproses kode ini:

- **Kompilasi (Compilation):** Seluruh kode program dibaca dan diubah menjadi file eksekusi (seperti `.exe`) sekaligus sebelum dijalankan. Contoh: C, C++, Go.
    
- **Interpretasi (Interpretation):** Kode dibaca dan dijalankan baris demi baris secara langsung saat program berjalan. Contoh: Python, Ruby.
    
- **Intermediate Code (Bytecode):** Beberapa bahasa (seperti Java) mengubah kode menjadi format tengah dulu, baru kemudian diterjemahkan lagi oleh mesin virtual (JVM).
    
- **Linker & Loader:** Tahap akhir di mana berbagai potongan kode digabungkan dan dimasukkan ke dalam memori RAM untuk dieksekusi oleh CPU.
    

## 4. Contoh Implementasi

Mari kita lihat perbedaan antara kode yang kita tulis dengan apa yang dilihat oleh "otak" komputer:

**Source Code (C++):**

C++

```
int x = 5 + 2;
```

**Machine Code (Biner - Representasi penyederhanaan):** Komputer tidak melihat kata `int` atau `x`. Ia melihat instruksi listrik seperti ini:

Plaintext

```
01010101  // Instruksi: Pindahkan angka ke register
00000101  // Data: Angka 5
00000010  // Data: Angka 2
00000001  // Instruksi: Tambahkan (ADD)
```

## 5. Koneksi dengan Web Development

Dalam **Web Development**, proses ini terjadi dalam beberapa lapisan:

1. **JavaScript di Browser:** Browser (seperti Chrome) memiliki mesin bernama **V8 Engine**. Mesin ini bertugas menerjemahkan JavaScript Anda secara instan (_Just-In-Time Compilation_) menjadi instruksi mesin agar animasi atau logika web Anda berjalan mulus.
    
2. **Transpilation:** Seringkali kita menulis kode modern (ES6+) yang tidak dimengerti browser lama. Kita menggunakan alat seperti **Babel** untuk menerjemahkan kode modern tersebut ke versi lama agar bisa diproses oleh semua "penerjemah" di berbagai browser.
    
3. **Performance:** Memahami cara komputer memproses kode membantu Anda menulis kode yang "ramah mesin", sehingga website Anda tidak lemot dan tidak memakan banyak memori RAM pengguna.
