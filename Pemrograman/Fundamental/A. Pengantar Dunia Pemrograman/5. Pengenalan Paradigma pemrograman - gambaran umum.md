## 1. Definisi Singkat

**Paradigma Pemrograman** adalah gaya, pendekatan, atau cara pandang mendasar dalam merancang dan menyusun kode untuk menyelesaikan masalah. Tiga paradigma yang paling umum digunakan adalah **Prosedural** (berbasis urutan langkah), **Object-Oriented / OOP** (berbasis objek), dan **Fungsional** (berbasis fungsi matematika murni).

## 2. Analogi: Membangun dan Memperbaiki Mobil

Bayangkan Anda sedang memproduksi dan merawat sebuah mobil:

- **Prosedural:** Seperti membaca **Buku Manual Perakitan**. Anda harus mengikuti instruksi langkah demi langkah dari atas ke bawah (1. Pasang rangka, 2. Pasang mesin, 3. Pasang ban). Jika ada satu langkah yang terlewat atau diubah di tengah jalan, seluruh proses bisa berantakan.
    
- **Object-Oriented Programming (OOP):** Seperti merancang mobil berdasarkan **Komponen (Objek)**. Ada objek "Mesin", objek "Pintu", dan objek "Roda". Masing-masing objek punya data (ukuran ban) dan perilaku (bisa berputar). Jika ban bocor, Anda cukup fokus memperbaiki "Objek Roda" tanpa perlu membongkar atau memikirkan "Objek Mesin".
    
- **Fungsional:** Seperti menggunakan **Mesin Cetak Otomatis** di pabrik. Anda memasukkan bahan mentah baja (input), mesin memprosesnya tanpa mengubah kondisi pabrik, dan selalu mengeluarkan pintu mobil yang sama persis (output). Mesin ini berfokus pada hasil yang konsisten dan meminimalkan "efek samping" (kerusakan mesin lain).
    

## 3. Poin-Poin Penting

- **Pemrograman Prosedural:**
    
    - Fokus pada urutan instruksi (Top-Down).
        
    - Data dan fungsi pengeksekusi data dibuat terpisah.
        
    - Sangat baik untuk skrip sederhana, tapi kodenya bisa menjadi "spaghetti" (sulit dilacak) pada aplikasi skala besar.
        
- **Object-Oriented Programming (OOP):**
    
    - Fokus pada _Objek_ yang menggabungkan data (_atribut/properties_) dan perilaku (_method_).
        
    - Memiliki pilar utama seperti _Encapsulation_ (menyembunyikan data internal) dan _Inheritance_ (pewarisan sifat ke objek lain), sehingga kode lebih mudah digunakan ulang (_reusable_).
        
- **Pemrograman Fungsional:**
    
    - Fokus pada evaluasi _Fungsi Murni_ (Pure Functions). Input yang sama harus selalu menghasilkan output yang sama.
        
    - Menghindari _Mutability_ (perubahan nilai variabel) dan _Side-effects_ (efek samping yang mengubah status program lain).
        
    - Sangat aman untuk pemrosesan data besar dan _concurrency_ (proses paralel).
        

## 4. Contoh Implementasi

Berikut adalah perbandingan sederhana menggunakan **JavaScript** untuk menghitung total harga barang:

**Prosedural (Langkah demi langkah):**

JavaScript

```
let harga = 10000;
let pajak = 2000;

function hitungTotal() {
    return harga + pajak; // Bergantung pada variabel di luar fungsi
}
console.log(hitungTotal());
```

**Object-Oriented (Berbasis Objek):**

JavaScript

```
class Barang {
    constructor(harga, pajak) {
        this.harga = harga;
        this.pajak = pajak;
    }
    hitungTotal() {
        return this.harga + this.pajak; // Data dan fungsi menyatu dalam objek
    }
}
const sepatu = new Barang(10000, 2000);
console.log(sepatu.hitungTotal());
```

**Fungsional (Fungsi murni tanpa efek samping):**

JavaScript

```
// Fungsi tidak mengubah variabel luar, hanya memproses input menjadi output
const hitungTotalFungsional = (hargaBarang, nilaiPajak) => hargaBarang + nilaiPajak;

console.log(hitungTotalFungsional(10000, 2000));
```

## 5. Koneksi dengan Web Development

Dalam **Web Development** modern, kita sering mencampur ketiga paradigma ini (_Multi-paradigm_):

1. **OOP di Backend:** Jika Anda membangun server atau API menggunakan bahasa seperti PHP (Laravel), Java (Spring), atau Python (Django), Anda hampir pasti menggunakan OOP (arsitektur _Model-View-Controller_). Struktur OOP sangat rapi untuk mengelola sistem _database_ dan _user_ yang kompleks.
    
2. **Fungsional di Frontend:** Saat ini, _framework_ populer seperti **React.js** sangat mendorong penggunaan gaya Fungsional. Pembuatan antarmuka web dilakukan menggunakan _Functional Components_ karena UI (User Interface) web lebih mudah diprediksi dan diuji jika tidak banyak status (state) yang berubah sembarangan.
    
3. **Prosedural untuk Scripting:** Terkadang, developer web masih menggunakan cara prosedural untuk _script_ otomatisasi sederhana, misalnya kode pendek untuk membuat menu navigasi (_hamburger menu_) bisa diklik.
